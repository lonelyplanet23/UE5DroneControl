# 多机地图交互与阵列控制架构改造清单

## 1. 文档目的

本文档用于把《多机地图交互与阵列控制需求文档》中的 5 个重点需求，拆解为可实施的架构改造清单。

本文档重点回答 4 个问题：

- 需要新建哪些 C++ 类、蓝图、Widget、数据资产和输入资产
- 需要改动哪些现有类、配置和脚本
- 每个新增/改动项的输入、输出、核心数据结构是什么
- 5 个需求之间哪些能力应共享，避免后续重复造轮子

---

## 2. 当前项目现状

当前工程已经具备以下基础：

- 已启用 `CesiumForUnreal` 插件
- 已有 `AUE5DroneControlCharacter` 负责本地玩家无人机控制和 UDP 发送
- 已有 `ARealTimeDroneReceiver` 负责 YAML 遥测接收、NED 到 UE 坐标转换和模型驱动
- 已有 `AMultiDroneManager`、`AMultiDronePlayerController`、`AMultiDroneCharacter` 作为多机原型
- 已有 `Variant_Strategy` 样板，可复用其 RTS 式相机、选择和 HUD 思路

当前主要问题：

- 运行架构仍以 `TopDown` 模板为核心，业务状态没有独立的多机层
- 相机、输入、控制、通信、选中状态、UI 状态分散在多个 Actor 内
- 多机能力目前主要停留在“切换镜头/路由发送者”，还没有统一注册表、统一状态模型和统一 UI
- 坐标转换逻辑主要放在接收器中，尚未沉淀为全局服务
- 阵列、信息面板、地图状态、自由镜头等需求还没有共享底座

结论：

- 不建议继续把 5 个 FR 直接叠加在 `BP_TopDownCharacter` / `BP_TopDownController` 上
- 建议新增一层 `DroneOps` 业务架构，作为正式运行入口

---

## 3. 目标分层架构

建议按以下 6 层组织：

1. 地图层

- 负责 Cesium 场景初始化、地图加载状态、地图坐标锚点和世界对齐

2. 无人机注册与状态层

- 负责维护 `DroneId -> Sender / Receiver / Telemetry / Selection / LockState`

3. 输入与交互层

- 负责点击无人机、点击地图、中键信息面板、F 键自由镜头、多选和阵列播放控制

4. 相机层

- 负责“跟随镜头”和“自由镜头”的切换、回退和过渡

5. 执行与调度层

- 负责单机目标点下发、阵列序列播放、控制抢占和暂停/停止

6. 展示层

- 负责地图加载状态、当前控制对象、无人机信息面板、镜头模式和阵列状态显示

## 5. 新建资产总表

### 5.1 共享 C++ 资产

### `DroneOpsTypes.h`

- 类型：共享枚举与结构体头文件
- 用途：承载 5 个 FR 的统一数据模型
- 输入：控制请求、遥测快照、阵列配置、镜头状态
- 输出：供 Controller、Subsystem、Widget、数据资产统一引用
- 核心数据结构：
- `EDroneCameraMode`：`Follow` / `Free`
- `EDroneAvailability`：`Online` / `Offline` / `Lost`
- `EDroneControlLockReason`：`None` / `FormationPlayback` / `Offline` / `InvalidTarget`
- `FDroneDescriptor`：`DroneId`、`DisplayName`、`DroneIndex`、`ThemeColor`、`ReceivePort`、`SendPort`、`RosTopicPrefix`
- `FDroneTelemetrySnapshot`：`DroneId`、`Availability`、`WorldLocation`、`NedLocation`、`Velocity`、`Attitude`、`Altitude`、`LastUpdateTime`
- `FDroneTargetCommand`：`DroneId`、`TargetWorldLocation`、`TargetNedLocation`、`Mode`、`IssuedAt`
- `FCameraModeState`：`CameraMode`、`FollowDroneId`、`LastFollowLocation`、`LastFollowRotation`
- `FFormationMemberSlot`：`DroneId`、`RoleName`、`RelativeOffset`
- `FFormationStep`：`StepId`、`DurationSec`、`MemberTargets`
- `FFormationExecutionState`：`PresetId`、`StepIndex`、`ElapsedSec`、`bPaused`、`ParticipantIds`

### `DroneRegistrySubsystem.h/.cpp`

- 类型：`UGameInstanceSubsystem`
- 用途：全局无人机注册表和状态中心
- 输入：无人机注册、遥测更新、选中切换、控制锁定、相机跟随目标变更
- 输出：查询接口和广播事件
- 对外接口建议：
- `RegisterSenderPawn(DroneId, PawnRef)`
- `RegisterReceiverActor(DroneId, ReceiverRef)`
- `UpdateTelemetry(DroneId, Snapshot)`
- `SetPrimarySelectedDrone(DroneId)`
- `SetMultiSelectedDrones(DroneIds)`
- `ApplyControlLock(DroneId, LockReason)`
- `GetPrimarySelectedDrone()`
- `GetTelemetry(DroneId)`
- `GetAllDroneDescriptors()`
- 广播事件建议：
- `OnDroneRegistered`
- `OnPrimarySelectionChanged`
- `OnTelemetryUpdated`
- `OnControlLockChanged`

### `DroneCoordinateService.h/.cpp`

- 类型：坐标转换服务类
- 用途：统一处理 `Cesium 地理坐标`、`UE 世界坐标`、`PX4 NED 坐标`
- 输入：地图点击位置、无人机遥测、阵列偏移、地图锚点配置
- 输出：供 FR-01、FR-02、FR-05 复用的标准转换结果
- 核心数据：
- 地图原点
- 北向对齐规则
- 高度基准
- Cesium 地理锚点配置
- 对外接口建议：
- `WorldToNed(FVector WorldLocation)`
- `NedToWorld(FVector NedLocation)`
- `WorldToGeographic(FVector WorldLocation)`
- `GeographicToWorld(FVector GeoLocation)`

### `DroneSelectionComponent.h/.cpp`

- 类型：无人机交互组件
- 用途：承载单选/多选/高亮状态
- 输入：点击命中、拖框命中、清空选择、阵列编辑选择
- 输出：高亮状态、UI 可读的选择状态
- 核心数据：
- `DroneId`
- `bIsPrimarySelected`
- `bIsSecondarySelected`
- `SelectionThemeColor`

### `DroneTelemetryComponent.h/.cpp`

- 类型：遥测状态组件
- 用途：把 `ARealTimeDroneReceiver` 解析出的实时数据抽象成可绑定状态
- 输入：YAML 遥测解析结果
- 输出：Widget、状态栏、信息面板可直接读取的快照
- 核心数据：
- `CurrentSnapshot`
- `LastOnlineTime`
- `bHasValidTelemetry`

### `DroneCommandSenderComponent.h/.cpp`

- 类型：控制发送组件
- 用途：把目标点/模式切换从角色类中拆出
- 输入：`FDroneTargetCommand`
- 输出：UDP 控制包
- 核心数据：
- `RemoteIP`
- `RemotePort`
- `LastIssuedCommand`
- `SendInterval`
- 对外接口建议：
- `SendTargetCommand(Command)`
- `SendHoverCommand(DroneId)`

### `DroneFreeCameraPawn.h/.cpp`

- 类型：自由镜头 Pawn
- 用途：FR-04 的正式镜头载体
- 输入：`W/A/S/D`、`Q/E`、鼠标 Look、初始位置/旋转
- 输出：相机变换、镜头状态
- 核心数据：
- `MoveSpeed`
- `Acceleration`
- `LookSensitivity`
- `CurrentVelocity`
- 关键要求：
- `W/A/S/D` 必须相对于当前视角方向移动
- `Q/E` 控制垂直升降

### `DroneOpsPlayerController.h/.cpp`

- 类型：正式业务控制器
- 用途：统一管理 FR-02、FR-03、FR-04、FR-05 输入与状态机
- 输入：点击、悬停、中键、F 键、阵列播放控制、自由镜头移动
- 输出：选中状态变更、目标点派发、信息面板打开、自由镜头切换、阵列调度
- 核心数据：
- `PrimarySelectedDroneId`
- `MultiSelectedDroneIds`
- `HoveredDroneId`
- `CameraModeState`
- `FreeCamRef`
- `TargetMarkerRef`
- 不建议继续直接堆在现有 `AMultiDronePlayerController` 上

### `DroneOpsGameMode.h/.cpp`

- 类型：正式业务 GameMode
- 用途：切换到新的 DroneOps 运行入口
- 输入：默认 Pawn、PlayerController、HUD、地图初始化配置
- 输出：统一业务框架类

### `FormationPresetDataAsset.h/.cpp`

- 类型：`UPrimaryDataAsset`
- 用途：存储内置阵列预设和自定义序列结构
- 输入：编辑器配置、运行时载入
- 输出：阵列播放系统可执行的步骤列表
- 核心数据：
- `PresetId`
- `DisplayName`
- `PreviewText`
- `Steps`
- `DefaultDuration`
- `Tags`

### `FormationPlaybackSubsystem.h/.cpp`

- 类型：阵列播放子系统
- 用途：执行 FR-05 的步骤调度和锁控
- 输入：选中无人机集、阵列预设、播放/暂停/停止命令
- 输出：每个参与无人机的目标点、阵列状态广播
- 核心数据：
- `ExecutionState`
- `CurrentPreset`
- `ControlledDroneIds`
- `ConflictPolicy`
- 对外接口建议：
- `StartPlayback(Preset, DroneIds)`
- `PausePlayback()`
- `ResumePlayback()`
- `StopPlayback()`

### `DroneOpsSaveGame.h/.cpp`

- 类型：`USaveGame`
- 用途：保存自定义阵列序列
- 输入：用户编辑后的阵列配置
- 输出：本地保存与加载结果
- 核心数据：
- `SavedPresets`
- `LastEditedPresetId`

### `DroneSelectableInterface.h/.cpp`

- 类型：C++ Interface
- 用途：统一无人机被点击、被悬停、被高亮时的交互接口
- 输入：`OnPrimarySelected`、`OnHovered`、`OnDeselected`
- 输出：Actor 高亮、描边、状态标识切换

### `DroneInfoProviderInterface.h/.cpp`

- 类型：C++ Interface
- 用途：信息面板读取无人机状态的统一入口
- 输入：面板请求或 HUD 请求
- 输出：`FDroneTelemetrySnapshot`、扩展字段字典、状态文本

## 6. 现有文件改动总表

### `Config/DefaultEngine.ini`

- 改动原因：当前默认地图和默认 GameMode 仍指向 `TopDown`
- 需要改动：
- `GameDefaultMap` 切到 `LVL_DroneOps_Cesium`
- `EditorStartupMap` 切到 `LVL_DroneOps_Cesium`
- `GlobalDefaultGameMode` 切到 `BP_DroneOpsGameMode`

### `Config/DefaultInput.ini`

- 改动原因：当前只有 `空格/0/1/W/S` 等旧输入
- 需要改动：
- 保留旧映射仅作兼容
- 将正式交互迁移到 `Enhanced Input`
- 若仍保留 Legacy，则补 `F / Q / E / 中键 / 多机切换` 的兜底映射

### `Source/UE5DroneControl/UE5DroneControl.Build.cs`

- 改动原因：未来 UI、存档、配置和可能的 Cesium C++ 调用会增加模块依赖

### `Source/UE5DroneControl/UE5DroneControlCharacter.h/.cpp`

## 9. 最终建议

本项目最关键的不是“多写几个蓝图”，而是先把共享业务层建立起来。

如果继续沿用当前做法，把点击、相机、目标点、遥测、信息面板、阵列都堆在 `TopDownCharacter`、`PlayerController` 和 `RealTimeDroneReceiver` 中，后续 5 个需求会互相耦合，维护成本会快速上升。

推荐路线：

- 新建 `DroneOps` 正式业务层
- 保留 `TopDown` 和当前 `MultiDrone*` 作为兼容或原型层
- 用“注册表 + 坐标服务 + 控制发送组件 + 遥测组件 + 正式 HUD”作为长期底座
