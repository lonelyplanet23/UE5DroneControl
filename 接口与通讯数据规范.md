> 版本：v1.0

> 目的：梳理并统一各文档中的接口定义和通讯数据格式，解决已发现的冲突

## 一、系统通讯总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        UE5 前端                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │         UE 通讯模块（HTTP Client + WebSocket Client）      │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
        │ HTTP REST（注册/查询/删除/锚点/阵列）
        │ WebSocket（控制指令↓ 遥测+事件+告警↑）
┌─────────────────────────────────────────────────────────────────┐
│                     后端服务（C++）                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌────────────────┐  │
│  │ 注册管理  │  │ 队列管理  │  │ 执行引擎  │  │  坐标转换模块  │  │
│  └──────────┘  └──────────┘  └──────────┘  └────────────────┘  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │          多线程 UDP 发送（每架无人机独立线程）              │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
        │ UDP 24字节控制包（后端→无人机，≥2Hz 心跳，每机独立线程）
        │ UDP YAML 遥测包（无人机→后端，10Hz）
┌─────────────────────────────────────────────────────────────────┐
│                     PX4 无人机（×6 max）                          │
└─────────────────────────────────────────────────────────────────┘
```

## 二、HTTP REST 接口

### 2.1 无人机管理

#### GET /api/drones — 获取所有无人机列表

响应：
```json
{
  "drones": [
    {
      "id": "d1",
      "name": "侦察机-1",
      "model": "DJI M300",
      "slot": 1,
      "ip": "192.168.30.104",
      "port": 8889,
      "video_url": "rtsp://...",
      "status": "online",
      "battery": 85
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 后端自动分配的唯一 ID |
| name | string | 用户自定义名称（不可重复） |
| model | string | 无人机型号 |
| slot | int | 编号（1~6），对应 UDP 端口映射，注册时手动指定 |
| ip | string | 无人机通讯 IP |
| port | int | 无人机通讯端口 |
| video_url | string | 视频流地址（可选） |
| status | string | 连接状态：`offline` / `connecting` / `online` / `lost` |
| battery | int | 电量百分比，-1 表示未知 |

#### POST /api/drones — 注册新无人机

请求体：
```json
{
  "name": "侦察机-1",
  "model": "DJI M300",
  "slot": 1,
  "ip": "192.168.30.104",
  "port": 8889,
  "video_url": "rtsp://..."
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| name | 是 | 名称，不可与已注册无人机重复 |
| model | 是 | 型号 |
| slot | 是 | 编号 1~6，不可与已注册编号重复 |
| ip | 是 | 无人机通讯 IP |
| port | 是 | 无人机通讯端口 |
| video_url | 否 | 视频流地址 |

响应（成功 201）：
```json
{ "id": "d1", "slot": 1 }
```

错误响应：
- 400：字段缺失或格式错误
- 409：name 或 slot 已被占用
- 507：已达最大注册数量（6 架）

#### PUT /api/drones/{id} — 更新无人机信息

请求体：与 POST 相同（除 id 外所有字段可选，仅传需要修改的字段）。

响应（成功 200）：
```json
{ "id": "d1", "updated": true }
```

#### DELETE /api/drones/{id} — 删除无人机

响应（成功 200）：
```json
{ "id": "d1", "deleted": true }
```

错误响应：
- 404：ID 不存在

#### GET /api/drones/{id}/anchor — 获取 GPS 锚点

响应（成功 200）：
```json
{
  "drone_id": "d1",
  "gps_lat": 39.9042,
  "gps_lon": 116.4074,
  "gps_alt": 50.0
}
```

错误响应：
- 404：ID 不存在
- 425：无人机尚未上电（无锚点数据）

> 说明：此接口为补偿查询，正常流程中 GPS 锚点通过 WebSocket `power_on` 事件直接推送。

### 2.2 阵列任务

#### POST /api/arrays — 下发阵列任务

请求体：
```json
{
  "array_id": "a1",
  "mode": "recon",
  "paths": [
    {
      "pathId": 1,
      "drone_id": "d1",
      "bClosedLoop": false,
      "waypoints": [
        {
          "location": { "x": 0, "y": 0, "z": 0 },
          "segmentSpeed": 0,
          "waitTime": 0
        },
        {
          "location": { "x": 200, "y": 0, "z": 227.02 },
          "segmentSpeed": 1,
          "waitTime": 0
        },
        {
          "location": { "x": 400, "y": 0, "z": 144.04 },
          "segmentSpeed": 2,
          "waitTime": 0
        }
      ]
    },
    {
      "pathId": 2,
      "drone_id": "d2",
      "bClosedLoop": false,
      "waypoints": [
        {
          "location": { "x": 0, "y": 100, "z": 0 },
          "segmentSpeed": 0,
          "waitTime": 0
        },
        {
          "location": { "x": 200, "y": 124.92, "z": 151.46 },
          "segmentSpeed": 15,
          "waitTime": 0
        }
      ]
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| mode | string | 执行模式：`recon`（侦察）/ `patrol`（巡逻）/ `attack`（攻击） |
| paths | array | 路径列表，每条路径对应一架无人机 |
| paths[].pathId | int | 路径编号（唯一标识） |
| paths[].drone_id | string | 执行该路径的无人机 ID |
| paths[].bClosedLoop | bool | 是否闭环循环（attack 模式下强制为 false） |
| paths[].waypoints | array | 航点序列 |
| waypoints[].location | object | 航点坐标，x/y/z 为相对锚点的 UE 偏移（厘米） |
| waypoints[].segmentSpeed | float | 该航段飞行速度（m/s），0 表示使用默认速度 |
| waypoints[].waitTime | float | 到达该航点后的悬停等待时间（秒），0 表示不等待 |

> 航点坐标说明：location 中的 x/y/z 使用 UE 坐标偏移（厘米），与 move 指令坐标系一致。后端收到后统一转换为 NED。

响应（成功 201）：
```json
{ "array_id": "a1", "status": "assembling" }
```

#### POST /api/arrays/{id}/stop — 停止阵列任务

响应（成功 200）：
```json
{ "array_id": "a1", "status": "stopped" }
```

## 三、WebSocket 消息协议

连接地址：`ws://{backend_host}:{ws_port}/`（默认端口 8081）

所有消息均为 JSON 格式，以 `type` 字段区分消息类型。

### 3.1 UE5 → 后端（控制指令）

#### move — 目标点指令

```json
{
  "type": "move",
  "drone_id": "d1",
  "x": 1000.0,
  "y": 2000.0,
  "z": 500.0
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| drone_id | string | 目标无人机 ID |
| x / y / z | float | 相对锚点的 UE 坐标偏移（厘米） |

> 不含 mode 字段。move 语义即"移动到目标点"，后端自动以 Mode=1 发送 UDP 控制包。悬停通过 pause 指令实现。

#### pause — 暂停

```json
{
  "type": "pause",
  "drone_ids": ["d1", "d2"]
}
```

后端收到后，向对应无人机发送 Mode=0（悬停）的 UDP 控制包，并暂停指令队列推进。

#### resume — 恢复

```json
{
  "type": "resume",
  "drone_ids": ["d1", "d2"]
}
```

### 3.2 后端 → UE5（推送）

#### telemetry — 遥测数据

```json
{
  "type": "telemetry",
  "drone_id": "d1",
  "x": 10000.0,
  "y": 20000.0,
  "z": 5000.0,
  "yaw": 45.0,
  "pitch": 0.0,
  "roll": 0.0,
  "speed": 3.5,
  "battery": 85
}
```

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| x / y / z | float | 厘米 | 相对锚点的 UE 坐标偏移（后端已完成 NED→UE 转换） |
| yaw | float | 度 | UE5 Yaw 角（已翻转，Yaw_UE = -Yaw_NED） |
| pitch | float | 度 | 俯仰角 |
| roll | float | 度 | 横滚角 |
| speed | float | m/s | 标量速度（由 velocity 向量求模：`sqrt(vN² + vE² + vD²)`） |
| battery | int | % | 电量百分比，-1 表示数据不可用 |

> 推送频率：跟随遥测接收频率（约 10Hz）。

#### event — 状态事件

```json
// 无人机上电（携带 GPS 锚点）
{
  "type": "event",
  "drone_id": "d1",
  "event": "power_on",
  "gps_lat": 39.9042,
  "gps_lon": 116.4074,
  "gps_alt": 50.0
}

// 断联重连（携带新 GPS 锚点）
{
  "type": "event",
  "drone_id": "d1",
  "event": "reconnect",
  "gps_lat": 39.9043,
  "gps_lon": 116.4075,
  "gps_alt": 50.5
}

// 失联
{
  "type": "event",
  "drone_id": "d1",
  "event": "lost_connection"
}
```

| event 值 | GPS 字段 | 说明 |
|-----------|----------|------|
| power_on | 必须携带 | 首次收到遥测，UE5 据此建立锚点 |
| reconnect | 必须携带 | NED 原点已重置，UE5 需更新锚点 |
| lost_connection | 无 | 超时未收到遥测（默认 10s） |

#### assembling — 集结进度

```json
{
  "type": "assembling",
  "array_id": "a1",
  "ready_count": 2,
  "total_count": 4
}
```

#### assembly_complete — 集结完成

```json
{
  "type": "assembly_complete",
  "array_id": "a1"
}
```

#### assembly_timeout — 集结超时

```json
{
  "type": "assembly_timeout",
  "array_id": "a1",
  "ready_count": 2,
  "total_count": 4
}
```

#### alert — 告警

```json
// 低电量
{
  "type": "alert",
  "drone_id": "d1",
  "alert": "low_battery",
  "value": 18
}

// 断联告警
{
  "type": "alert",
  "drone_id": "d1",
  "alert": "lost_connection"
}
```

> 告警去重：同一无人机同类告警触发后，状态未恢复前不重复推送。电量恢复到阈值以上后重置，下次低于阈值时重新触发。

## 四、UDP 通讯协议（后端 ↔ PX4 无人机）

### 4.1 控制包（后端 → 无人机）— 24 字节

```c
#pragma pack(push, 1)
struct DroneControlPacket {
    double   Timestamp;   // 8 bytes，Unix 时间戳（秒）
    float    X;           // 4 bytes，NED North（米）
    float    Y;           // 4 bytes，NED East（米）
    float    Z;           // 4 bytes，NED Down（米）
    uint32_t Mode;        // 4 bytes，0=悬停，1=移动
};
#pragma pack(pop)
// 总计 24 字节，小端序（Little-Endian）
// Python: struct.pack('<dfffI', timestamp, x, y, z, mode)
```

| 字段 | 偏移 | 大小 | 类型 | 说明 |
|------|------|------|------|------|
| Timestamp | 0 | 8 | double | Unix 时间戳（秒） |
| X | 8 | 4 | float | NED North 偏移（米） |
| Y | 12 | 4 | float | NED East 偏移（米） |
| Z | 16 | 4 | float | NED Down 偏移（米） |
| Mode | 20 | 4 | uint32 | 0=悬停（心跳），1=移动 |

发送频率：
- 心跳包（Mode=0）：≥2Hz，防止 PX4 退出 Offboard 模式
- 移动包（Mode=1）：指令派发时高频发送

### 4.2 遥测包（无人机 → 后端）— YAML 格式

后端通过 drone_data_bridge.py 从 ROS2 话题获取，解析以下字段：

```yaml
timestamp: 1713800000000000   # uint64，微秒
position: [1.0, 2.0, -10.0]  # NED [North, East, Down]，米
q: [1.0, 0.0, 0.0, 0.0]      # 四元数 [w, x, y, z]
velocity: [0.5, 0.3, -0.1]   # NED [vN, vE, vD]，m/s
angular_velocity: [0.0, 0.0, 0.01]  # rad/s
```

| 字段 | 坐标系 | 单位 | 说明 |
|------|--------|------|------|
| timestamp | — | 微秒 | PX4 系统时间戳 |
| position | NED | 米 | 相对上电原点的位置偏移 |
| q | FRD 机体系 | — | 四元数，YAML 顺序 [w, x, y, z] |
| velocity | NED | m/s | 速度向量 |
| angular_velocity | — | rad/s | 角速度 |

> 注意：YAML 遥测包不含 battery 字段。电量数据需后端额外订阅 ROS2 话题 `/px4_N/fmu/out/battery_status` 获取。

### 4.3 多机端口映射

| 编号 (slot) | 遥测接收端口 (recv) | 控制发送端口 (send) | ROS2 话题前缀 |
|-------------|---------------------|---------------------|---------------|
| 1 | 8888 | 8889 | /px4_1 |
| 2 | 8890 | 8891 | /px4_2 |
| 3 | 8892 | 8893 | /px4_3 |
| 4 | 8894 | 8895 | /px4_4 |
| 5 | 8896 | 8897 | /px4_5 |
| 6 | 8898 | 8899 | /px4_6 |

> 当前项目仅支持 3 架无人机，编号 4~6 暂不启用。

## 六、关键流程时序

### 6.1 无人机上电锚定

```
UE5                    后端                    PX4
 │                      │                       │
 │                      │←── YAML遥测（首包）────│
 │                      │  记录GPS锚点            │
 │←── WS: power_on ────│  （携带gps_lat/lon/alt）│
 │  Cesium转UE世界坐标   │                       │
 │  设置AnchorWorldLoc   │                       │
 │                      │←── YAML遥测（持续）────│
 │←── WS: telemetry ───│  NED→UE偏移转换        │
 │  镜像机 = 锚点+偏移   │                       │
```

### 6.2 实时控制

```
UE5                    后端                    PX4
 │  用户点击地图          │                       │
 │  target - Anchor      │                       │
 │──── WS: move ────────→│                       │
 │                      │  UE偏移→NED转换         │
 │                      │──── UDP 24B ──────────→│
 │                      │  心跳（≥2Hz）           │
 │                      │──── UDP 24B ──────────→│
 │←── WS: telemetry ───│←── YAML遥测 ──────────│
 │  更新镜像机位置        │                       │
```

### 6.3 断联重连

```
UE5                    后端                    PX4
 │                      │  10s无遥测              │
 │←── WS: event ───────│  event=lost_connection  │
 │←── WS: alert ───────│  alert=lost_connection  │
 │  显示断联告警弹窗      │                       │
 │                      │                       │
 │                      │←── YAML遥测（重连首包）─│
 │                      │  记录新GPS锚点          │
 │←── WS: event ───────│  event=reconnect        │
 │                      │  （携带新gps_lat/lon/alt）│
 │  重新Cesium转换       │                       │
 │  更新AnchorWorldLoc   │                       │
```

## 七、配置参数汇总

### 网络部署说明

UE5 前端和后端服务运行在**同一局域网的两台不同机器**上：

| 角色 | 机器 | 说明 |
|------|------|------|
| UE5 前端 | Windows PC（运行 UE5） | HTTP Client + WebSocket Client，主动连接后端 |
| 后端服务 | 独立机器（运行 C++ 后端） | HTTP Server + WebSocket Server，监听局域网 IP |

UE5 端需在配置中填写后端机器的**局域网 IP**（如 `192.168.x.x`），而非 `127.0.0.1`。

接口地址示例：
- HTTP：`http://192.168.x.x:8080/api/drones`
- WebSocket：`ws://192.168.x.x:8081/`

后端服务需监听 `0.0.0.0`（所有网卡），而非 `127.0.0.1`，否则局域网内其他机器无法访问。

### 参数列表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| backend_host | （部署时填写） | 后端机器局域网 IP，UE5 端配置项 |
| http_port | 8080 | HTTP REST 服务端口 |
| ws_port | 8081 | WebSocket 服务端口 |
| max_count | 6 | 最大注册无人机数量 |
| heartbeat_hz | 2 | 心跳发送频率（Hz），不得低于 2 |
| lost_timeout_sec | 10 | 超时失联阈值（秒） |
| arrival_threshold_m | 1.0 | 到达航点的位置误差阈值（米） |
| assembly_timeout_sec | 60 | 集结超时阈值（秒） |
| avoidance_radius_m | 3.0 | 实时避障安全距离（米） |
| low_battery_threshold | 20 | 低电量告警阈值（%） |
