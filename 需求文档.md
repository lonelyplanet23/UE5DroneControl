# UE5DroneControl 外包需求文档

> 版本：v0.4（草稿）
> 日期：2026-04-08
> 面向：UE5 外包工程师

---

## 一、项目背景

本项目是一套基于 UE5 + PX4 的多无人机实时控制与可视化系统。UE5 作为操控台前端，通过后端服务与真实 PX4 无人机通信。

**技术约束**：
- UE5 版本：5.3（需支持后续替换 Cesium 为 SuperMap 插件）
- 地图插件：当前使用 CesiumForUnreal
- 最大同时控制无人机数量：100 架 
- 后端语言：非 Python，使用高性能语言重新开发（Go / Rust / C++ 等，具体由后端团队确定）

---

## 一附、实物无人机特性说明

> 本节供 UE5 工程师了解真实无人机的通信特性和约束，以便正确设计 UE5 端的控制逻辑和镜像机渲染。

### A.1 控制指令格式

UE5 向后端（或当前桥接层）发送控制指令，当前采用 UDP 小端字节序数据包：

**桥接层转发给 PX4 的单机包（24 字节）**：

```c
struct DroneSocketData {
    double  Timestamp;   // 8 bytes，Unix 时间戳（秒）
    float   X;           // 4 bytes，相对odometry原点 X （米）
    float   Y;           // 4 bytes，相对odometry原点 Y （米）
    float   Z;           // 4 bytes，相对odometry原点 Z （米）
    int32   Mode;        // 4 bytes，0=悬停，1=移动
};
// Python 解析：struct.unpack('<dfffi', data)
```

**当前限制**：Mode 字段仅支持 0（悬停）和 1（移动），**不支持指定速度**。速度控制为未来扩展功能，当前 UE5 端发送的速度信息不会被无人机执行，无人机以自身默认速度飞行。

其中坐标（x,y,z）是无人机相对于上电后原点的偏移量，NED坐标系，单位米

### A.2 指令频率要求

- （无人机 to 后端）心跳包：**10 Hz**（每 0.1 秒发送一次，维持 Offboard 模式）
- （后端 to 无人机）点击派发时：高频发送目标点
- （后端 to 无人机）PX4 Offboard 模式要求：控制指令频率 ≥ **2 Hz**，否则无人机自动退出 Offboard 模式并悬停

### A.3 遥测数据来源（odometry 话题）

无人机遥测数据通过 ROS2 话题 `/px4_N/fmu/out/vehicle_odometry` 发布，经桥接脚本转为 YAML 格式通过 UDP 发送给 UE5。

**YAML 数据字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | uint64 | 时间戳（微秒） |
| `position` | float[3] | NED 坐标（米），[North, East, Down] |
| `q` | float[4] | 四元数，YAML 中顺序为 [w, x, y, z]，UE5 中重排为 FQuat(x, y, z, w) |
| `velocity` | float[3] | NED 速度（m/s） |
| `angular_velocity` | float[3] | 角速度（rad/s） |

**重要**：`position` 字段的坐标原点是**无人机上电时的位置**，而非地图中心或任何固定地理坐标。这是 odometry 的固有特性。

### A.4 坐标系转换

**NED → UE5 世界坐标**（当前实现，`ARealTimeDroneReceiver` 中）：

```
UE5_X = NED_North × 100    (厘米)
UE5_Y = NED_East  × 100    (厘米)
UE5_Z = NED_Down  × (-100) (厘米，方向取反)
```

**四元数处理**：翻转 Z 分量（`-Q.Z`），翻转 Yaw（`-Yaw`）以适配 UE5 左手坐标系。

### A.5 镜像机定位策略（上电锚定 + odometry 偏移）

由于 odometry 坐标原点是无人机上电时的位置，镜像机在ue中世界定位采用以下策略：

1. **上电时锚定**：无人机上电后首次建立通信时，读取一次 GPS 坐标，将该 GPS 坐标转换为 Cesium 场景中的世界坐标，作为该无人机的**锚点位置**（`AnchorWorldLocation`）
2. **后续帧偏移**：之后每帧收到 odometry 数据，计算相对上电时刻的 NED 偏移量，叠加到锚点位置上，得到镜像机当前世界坐标：

```
MirrorWorldLocation = AnchorWorldLocation + NEDToUE5(CurrentNED - NED_at_power_on)
```

**断联重连处理**：无人机断联后重新上电，odometry 原点会重置（新的上电位置），锚点失效。重连后必须重新执行一次锚定流程（重新读取 GPS，重新设置锚点），实时部署模块中重连事件应触发镜像机位置重新对齐。

### A.6 坐标系约定速查

为避免 UE5 端、后端与 PX4 之间出现轴向、单位或正负方向混淆，实物无人机相关坐标系约定统一如下：

| 对象 | 坐标系/机体系 | 轴向定义 | 单位 | 备注 |
|------|---------------|----------|------|------|
| 无人机 odometry `position` | NED | X=North（北），Y=East（东），Z=Down（下） | 米 | 原点为无人机上电时位置 |
| 无人机姿态 `q` | FRD 机体系到参考系 | X=Forward（前），Y=Right（右），Z=Down（下） | 无 | YAML 顺序为 `[w, x, y, z]` |
| UE5 世界坐标 | UE World | X=Forward（前），Y=Right（右），Z=Up（上） | 厘米 | UE5 使用左手坐标系 |

**坐标映射关系**：

```
UE5_X = NED_North × 100
UE5_Y = NED_East  × 100
UE5_Z = NED_Down  × (-100)
```

反向发送控制点时同理：

```
NED_North = UE5_X × 0.01
NED_East  = UE5_Y × 0.01
NED_Down  = UE5_Z × (-0.01)
```

**实现含义**：

- odometry 的位置本质上是无人机局部 NED 偏移，不是地图地理坐标，也不是 UE5 世界原点下的绝对位置
- UE5 与 PX4 的 X/Y 轴语义基本一致，主要差异在于单位不同（厘米 vs 米）以及 Z 轴方向相反（Up vs Down）
- 姿态转换时除四元数分量重排外，还需要考虑 `Down → Up` 和 UE5 左手系带来的 Yaw 翻转

## 二、整体界面结构

打开 UE5 后，主界面分为三个功能模块，通过顶部 Tab 切换：

| 模块 | 功能概述 |
|------|----------|
| 阵列编辑 | 离线编辑无人机飞行阵列，支持预演和碰撞检测 |
| 无人机状态 | 注册、删除、查看无人机，与后端服务通信 |
| 实时部署 | 实时控制已注册无人机执行已编辑的阵列任务 |

**模块切换限制**：

- **实时部署模块**：当 UE 正在派发指令（实时控制或阵列执行进行中）时，**禁止切换到其他模块**。Tab/导航按钮置灰，点击时弹出提示："当前有任务正在执行，请先停止任务后再切换模块。"
- **阵列编辑模块**：存在未保存的编辑内容时，若用户尝试切换模块或关闭编辑界面，弹出警告："当前编辑尚未保存，包含以下未完成内容：[列出未完成项，如：未设置高度的航点 N 个、未保存的无人机配置 N 架]。是否放弃编辑并退出？"用户可选择"继续编辑"或"放弃并退出"。

## 三、阵列编辑模块

### 3.1 模块概述

阵列编辑模块用于离线编排无人机飞行任务序列，编辑结果保存为"阵列"，可在实时部署模块中执行。编辑过程在 Cesium 地图场景中进行。

### 3.2 阵列管理

#### 3.2.1 新建阵列

用户点击"新建阵列"按钮，弹出对话框，填写：
- **阵列名称**（必填，字符串，不可与已有阵列重名）

确认后进入阵列编辑界面。

#### 3.2.2 阵列列表

- 显示所有已保存的阵列，支持选中、重命名、删除
- 选中已有阵列可进入编辑或预演

### 3.3 无人机编辑流程

在阵列编辑界面中，用户按以下流程逐架添加无人机：

```
点击"新增无人机"
  → 选择无人机类型（从已注册无人机中选择）
  → 选择执行模式（巡逻 / 攻击 / 侦察）
  → 设置航点序列（见 3.4）
  → 完成后可继续点击"新增无人机"添加下一架
```

一个阵列中可包含多架无人机，每架无人机独立配置执行模式和航点序列。

### 3.4 执行模式说明

| 模式 | 行为描述 |
|------|----------|
| **侦察模式** | 无人机严格按照设定的航点序列依次飞行，不响应目标识别事件 |
| **巡逻模式** | 无人机按航点序列飞行，途中若在指定范围内识别到目标，则脱离序列飞向目标 |
| **攻击模式** | 无人机起飞后径直飞向唯一目标点，忽略中间航点 |

### 3.5 航点序列编辑

#### 3.5.1 俯视视角下设置航点坐标

- 进入航点编辑时，视角自动切换为**纯俯视（Top-Down）**
- 用户在 Cesium 地图上依次点击，每次点击添加一个航点
- 每个已添加的航点在地图上显示**光柱标记**（垂直光束，颜色可区分序号）
- 航点之间显示连线，表示飞行路径顺序
- 支持拖拽已有航点调整位置
- 支持删除单个航点或清空全部

#### 3.5.2 高度设置

- 设置完平面坐标后，进入**高度编辑模式**，视角切换为**自由镜头模式**（与实时部署模块共用，见第五章）
- 每个航点的高度独立设置，单位为**米（浮点数）**
- 高度输入方式：
  - **键盘输入**：选中航点后直接键入数字确认
  - **鼠标调节**：按住 `Alt` 键后鼠标上下拖拽调节高度（连续浮点数）
- 高度调节时，地图上的光柱高度实时更新，直观反映当前高度值

#### 3.5.3 路径速度设置

- 每段路径（相邻两个航点之间）可单独设置**移动速度**（可选）
- 若不设置，使用**系统默认速度**（全局可配置）
- 速度合法性检查：
  - 最小速度：> 0 m/s
  - 最大速度：由用户在系统设置中配置（全局上限）
  - 超出范围时显示错误提示，不允许保存

#### 3.5.4 轨迹闭合（循环模式）

- 当航点数量 ≥ 2 时，若用户点击的新航点与**已有航点重合**（或在一定容差范围内），弹出提示："是否设置为循环移动模式？"
- 确认后，最后一个航点与第一个航点之间自动连线，形成闭合轨迹
- 循环模式下无人机到达最后一个航点后自动返回第一个航点，无限循环
- 循环模式在地图上以闭合连线标注，与非循环路径视觉区分

### 3.6 阵列保存

- 点击"保存"按钮，将当前阵列持久化到本地
- **存储格式：JSON**（通过 UE5 `FJsonSerializer` 序列化 C++ 结构体），每个阵列保存为一个 `.json` 文件，文件名为阵列名称
- JSON 格式支持多维嵌套（阵列 → 无人机列表 → 航点列表），比 CSV 更适合表达层级数据，同时保持人类可读、可批量编辑

**坐标存储方式：相对坐标**

航点坐标以**相对偏移**形式存储，而非绝对 GPS 坐标：
- 每个阵列包含一个**阵列原点**（lat/lon/alt），自动取第一个航点为原点
- 各航点坐标存储为相对原点的 NED 偏移（北/东/下，单位：米）
- 这样同一个阵列可以在不同地理区域复用，只需在执行时重新指定原点

**C++ 内存结构**（对应 JSON 序列化目标）：

```cpp
struct FArrayWaypoint {
    float NorthOffset;   // 相对原点北向偏移（米）
    float EastOffset;    // 相对原点东向偏移（米）
    float AltitudeM;     // 绝对高度（米）
    float SpeedMps;      // 该段速度，0 表示使用默认速度
};

struct FArrayDroneSlot {
    int32 SlotIndex;
    EExecutionMode Mode;  // Patrol / Attack / Recon
    bool bLoop;
    TArray<FArrayWaypoint> Waypoints;
};

struct FDroneArray {
    FString Name;
    double OriginLat;
    double OriginLon;
    double OriginAlt;
    TArray<FArrayDroneSlot> Drones;
};
```

> 速度字段为 0 或缺省时表示使用系统默认速度。

### 3.7 阵列预演

保存后，可在阵列编辑界面点击"预演"按钮：

- 在 Cesium 场景中以动画形式模拟所有无人机按各自航点序列飞行
- 预演速度可调（1x / 2x / 4x）
- 支持暂停、继续、停止

#### 3.7.1 碰撞检测

预演过程中自动进行碰撞检测，分两类：

| 碰撞类型 | 检测内容 | 触发时机 |
|----------|----------|----------|
| 无人机间碰撞 | 任意两架无人机的飞行轨迹是否在时间和空间上重叠 | 预演实时检测 |
| 环境障碍碰撞 | 无人机轨迹是否与场景中的建筑、地形等障碍物相交 | 预演实时检测 |

碰撞发生时：
- 预演暂停
- 碰撞位置高亮显示（红色标记）
- 弹出提示，说明碰撞的无人机编号和发生时间点
- 用户可选择"忽略继续"或"返回编辑"

#### 3.7.2 避障说明

在实际执行（实时部署）阶段，无人机前往目标点的过程中应有**避障算法支持**（具体算法由后端/桥接层实现，UE 端负责展示避障状态）。

## 四、无人机状态模块

### 4.1 模块概述

无人机状态模块负责管理系统中所有已知无人机的注册信息和在线状态。该模块需要与**后端服务**通信，通信协议采用 **HTTP/REST**。

系统最多支持同时管理 **100 架**无人机。

### 4.2 通信架构

#### 4.2.1 UE ↔ 后端通信（HTTP/REST）

UE5 客户端通过 HTTP 请求与后端服务交互，用于无人机的注册管理和状态查询。

**接口列表（待后端确认）**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/drones` | GET | 获取所有已注册无人机列表 |
| `/api/drones` | POST | 注册新无人机 |
| `/api/drones/{id}` | DELETE | 删除指定无人机 |
| `/api/drones/{id}` | PUT | 更新指定无人机信息 |
| `/api/drones/{id}/status` | GET | 查询单架无人机在线状态 |
| `/api/drones/{id}/command` | POST | 向指定无人机发送控制命令（实时控制/阵列执行） |

**状态轮询**：UE 端定期（建议 2~5 秒）轮询 `/api/drones` 刷新在线状态显示。

#### 4.2.2 后端职责范围

后端不仅负责注册/状态管理，**同时接管控制命令转发**：

- UE 发送控制命令到后端（HTTP POST）
- 后端转发给对应的 Python 桥接层（multi_ue_controller.py → ue_to_px4_bridge.py → PX4）
- 后端维护每架无人机的连接状态和命令队列

> **待讨论**：实时控制和阵列执行过程中，UE 与后端的通讯频率和协议可能需要重新设计（HTTP 轮询延迟可能不满足实时控制需求，是否引入 WebSocket 或保留现有 UDP 链路作为控制通道）。后端与无人机的通讯是否沿用当前 Python 桥接方式，需进一步确认。

#### 4.2.3 后端 ↔ 无人机通信

后端与无人机之间的通信链路**需重新设计**，不再沿用现有 Python 桥接层。新后端使用高性能语言（Go / Rust / C++ 等）实现，直接管理与无人机的通信。

具体通信协议和链路架构待后端团队确认，需覆盖以下职责：
- 向各无人机转发控制命令（目标点、阵列指令、暂停等）
- 接收各无人机遥测数据（位置、姿态、电量、速度等）并转发给 UE 端
- 维护每架无人机的连接状态和心跳检测

> **待讨论**：后端与无人机的底层通信协议（UDP / MAVLink / ROS2 等）及链路拓扑需在后端开发启动前与需求方确认。

### 4.3 无人机注册

用户点击"注册无人机"按钮，弹出注册表单。注册表单的具体字段设计（包括必填项、字段类型、与后端通信协议的对应关系）列为**待讨论事项**（见七、6），需在后端接口确定后一并设计。

当前已知的基本字段方向：无人机名称、唯一 ID、型号、网络地址/端口、视频流地址（可选）。

**注册校验原则**（具体规则待后端确认）：
- 名称和 ID 系统内唯一
- 网络地址格式合法性校验
- 提交后向后端发送注册请求，成功后刷新无人机列表

> **无人机型号管理**：型号由用户在系统设置中自行添加，不预设固定枚举。最大速度上限为全局统一配置，不按型号区分。

### 4.4 无人机列表

注册成功的无人机以列表形式展示，每行显示：

| 字段 | 说明 |
|------|------|
| 名称 | 无人机显示名称 |
| ID | 唯一标识符 |
| 类型 | 无人机型号 |
| IP:端口 | 网络地址 |
| 在线状态 | 在线 / 离线 / 连接中 / 失联 |
| 操作 | 编辑、删除按钮 |

**在线状态说明**：
- **在线**：心跳正常，遥测数据持续更新
- **离线**：未建立连接或主动断开
- **连接中**：正在尝试建立连接
- **失联**：曾在线但超时未收到心跳（超时阈值可配置，建议默认 10 秒）

### 4.5 删除无人机

- 点击列表中的"删除"按钮
- 弹出确认对话框："确认删除无人机 [名称]？此操作不可撤销。"
- 确认后向后端发送 DELETE 请求
- 若该无人机当前处于**在线状态**或**正在执行任务**，显示警告："该无人机当前在线/执行中，强制删除可能导致失控，是否继续？"

### 4.6 编辑无人机信息

- 点击"编辑"按钮，弹出与注册相同的表单，预填当前信息
- 支持修改除 ID 以外的所有字段
- 修改后向后端发送更新请求

## 五、自由镜头模式（阵列编辑与实时部署共用）

> 本章描述自由镜头模式的交互规范，该模式在阵列编辑的高度设置阶段和实时部署模块中均会使用。

### 5.1 进入/退出自由镜头

- 按 `F` 键在**跟随镜头**和**自由镜头**之间切换
- 每次切换到自由镜头时，镜头初始位置和朝向与当前跟随镜头完全一致（无跳变）
- 切换时有平滑过渡动画
- 再次按 `F` 回到切换前所跟随的无人机的跟随镜头

### 5.2 自由镜头操作

| 输入 | 行为 |
|------|------|
| `W / A / S / D` | 相对当前镜头朝向的前后左右水平移动 |
| `Q` | 垂直下降 |
| `E` | 垂直上升 |
| 鼠标移动 | 控制镜头朝向（自由镜头下鼠标直接控制朝向，无需按住额外键） |
| 滚轮 | 调节移动速度 |

### 5.3 高度编辑时的自由镜头

在阵列编辑的高度设置阶段：
- 自由镜头模式自动激活
- **选中航点**：按住 `Alt` 键后，鼠标左键点击地图上的航点光柱即可选中该航点（未按 `Alt` 时鼠标左键不触发航点选中）
- **鼠标调节高度**：选中航点后，保持按住 `Alt`，鼠标上下拖拽调节高度（连续浮点数，单位：米）
- **键盘输入高度**：选中航点后，直接键入数字后按 `Enter` 确认高度
- 光柱高度实时更新，光柱一侧的UI以数值反映当前高度值
- **默认高度**：若某航点未手动设置高度，则使用系统默认高度（默认值在系统设置中统一配置，建议默认 50 米）；保存时未设置高度的航点自动填入默认值并提示用户

## 六、实时部署模块

### 6.1 模块概述

实时部署模块用于将已编辑的阵列下发给真实无人机执行，并实时监控执行状态。进入该模块后，场景中显示所有**已注册且通讯正常**的无人机。

模块内分为两种操作模式，通过 UI 顶部的 **Toggle 切换**：

| 模式 | 说明 |
|------|------|
| **实时控制模式** | 手动点击派发目标点，直接控制单架无人机 |
| **阵列执行模式** | 选择已保存阵列，批量下发给映射好的无人机执行 |

**模式切换规则**：
- 切换 Toggle 仅在**当前没有正在派发的指令**时可用
- 若当前有指令正在执行（无人机移动中、阵列运行中），Toggle 置灰，点击时提示："当前有指令正在执行，请先停止后再切换模式。"
- 停止所有指令（按 `P` 暂停或等待执行完毕）后，Toggle 恢复可用

### 6.2 无人机通讯状态判断

**通讯正常的判定方式**（可借鉴现有 `ARealTimeDroneReceiver` 的 `LastUpdateTime` 机制）：

- `ARealTimeDroneReceiver` 每次收到 UDP 遥测数据包时更新 `LastUpdateTime`
- 每帧计算 `TimeSinceLastUpdate = CurrentTime - LastUpdateTime`
- 若 `TimeSinceLastUpdate` 超过阈值（建议默认 **3 秒**，可配置），判定为通讯异常
- 通讯异常的无人机在实时部署模块中仍显示，但标记为"失联"状态，不可被选中执行任务

此机制与现有 `RealTimeDroneReceiver` 逻辑一致，可直接复用。

### 6.3 影子机与镜像机（双机策略）

实时部署模块中，每架真实无人机在 UE 场景中对应**两个虚拟实体**：

| 实体 | 名称 | 数据来源 | 说明 |
|------|------|----------|------|
| 影子机（Shadow Drone） | 指令机 | UE 端发出的控制指令 | 表示"无人机应该在哪里"，按照设定的速度和轨迹在 UE 中匀速插值移动 |
| 镜像机（Mirror Drone） | 遥测机 | 无人机传回的遥测数据 | 表示"无人机实际在哪里"，由 `ARealTimeDroneReceiver` 驱动，实时同步位置（XYZ）和偏转角（Yaw/Pitch/Roll） |

**理想状态**：影子机与镜像机位置重合，表示指令与实际执行一致。两者偏差可作为系统健康度指标。

**镜像机同步字段**：
- 世界坐标（X / Y / Z）——每帧更新，驱动渲染位置
- 偏转角（Yaw）、俯仰角（Pitch）、横滚角（Roll）——每帧更新，驱动渲染朝向
- 速度向量——不参与每帧渲染，仅在鼠标中键信息面板中展示（见 6.7）

#### 6.3.1 镜像机的世界定位

镜像机始终根据无人机传回的遥测数据，映射到 Cesium 场景中与现实世界地理位置对应的坐标上。这是镜像机的核心职责，与影子机无关：

- 遥测数据由 `ARealTimeDroneReceiver` 接收，经坐标转换后驱动镜像机位置
- 镜像机的位置始终代表"无人机在现实世界中的真实位置"，不受任何 UE 端指令影响
- 当前架构中，坐标转换沿用 `ARealTimeDroneReceiver` 的现有逻辑（NED → UE5 世界坐标），详见 `多机地图交互与阵列控制架构改造清单.md` 中的坐标系统设计

> **待讨论**（见七、7）：遥测数据的具体来源（话题名称、数据格式）、坐标系转换的精确实现（绝对坐标 vs 相对偏移、地图中心基准）、以及 UE5 端与后端之间遥测数据的传输格式，均需在后端架构确定后统一设计。

#### 6.3.2 进入实时部署模块时的影子机对齐

进入实时部署模块后，影子机需与镜像机（即真实位置）对齐：

- 系统根据镜像机当前位置，**自动将影子机移动到相同位置**，完成初始对齐
- 提供**手动触发对齐**按钮，允许用户在任意时刻重新对齐（例如无人机被手动移动后）
- 对齐过程中影子机有平滑插值动画，不发生瞬移

#### 6.3.3 视觉区分

- 影子机与镜像机使用不同的视觉样式加以区分（例如颜色、透明度、轮廓线）
- 具体视觉方案由 UE 工程师设计，需保证两者在重合时仍可区分

### 6.4 镜头系统

#### 6.4.1 默认镜头

进入实时部署模块后，默认为**跟随镜头**，跟随**第一架注册的无人机**（按注册顺序），视角为 45 度俯视跟随。

#### 6.4.2 切换跟随目标

- 键盘输入无人机 ID（数字键），镜头切换到对应无人机的跟随镜头
- 切换时有平滑过渡

#### 6.4.3 自由镜头切换

- 按 `F` 键切换到自由镜头（规则见第五章）
- 切换到自由镜头时，镜头从当前跟随位置开始
- 再次按 `F` 回到切换前所跟随的无人机

#### 6.4.4 跟随镜头下的高度调节

在**跟随镜头**且处于**实时控制模式**时：
- `Q / E` 键不移动镜头，而是**调节当前选中无人机的飞行高度**
- 调节量为相对当前高度的增减（单位：米，步长可配置）

### 6.5 实时控制模式

#### 6.5.1 选中无人机

- 左键点击，从镜头位置发射射线
- 若射线击中无人机 Mesh，则选中该无人机（击中多架时选择射线最先命中的一架）
- 若射线未击中任何无人机，此次点击无效（不清除已选中状态）

#### 6.5.2 派发目标点

- 选中无人机后，左键点击地图（Cesium 地形/建筑表面）
- 射线击中地图后，向选中无人机发送飞往该坐标的指令
- **高度策略**：飞行高度恒定（保持当前高度），若路径上存在障碍物则自动升高绕过，到达目标点上方后再下降至目标高度
- 目标点在地图上显示标记（颜色与无人机主题色一致）

#### 6.5.3 自由镜头下的操作

自由镜头模式下，鼠标移动控制镜头朝向，无法直接点击选中无人机或地图。需要：
- 按住 `Alt` 键：镜头固定（鼠标移动、WASDQE 均不移动镜头），此时鼠标可用于点击选中无人机或目标点
- 松开 `Alt`：恢复自由镜头控制

#### 6.5.4 暂停所有移动

- 按 `P` 键：所有无人机**立即停止移动**（悬停在当前位置）
- 再次按 `P` 或发送新指令可恢复

### 6.6 阵列执行模式

#### 6.6.1 执行前提条件

阵列执行需满足以下全部条件：

| 条件 | 说明 |
|------|------|
| 存在可执行阵列 | 阵列编辑模块中已保存至少一个阵列 |
| 无人机数量充足 | 阵列所需无人机数 ≤ 当前在线注册无人机数 |
| 完成无人机映射 | 阵列中每个无人机槽位已与一架真实无人机对应 |
| 已完成预演（建议） | 该阵列在阵列编辑模块中已预演通过 |

若未完成预演，系统弹出提示：

> "当前阵列尚未预演，为避免碰撞可能调整部分序列目标点高度。执行过程中将启用实时避障算法。是否继续？"

用户可选择继续或返回预演。**未预演执行时，UE 端需启用实时避障算法**（算法在 UE 侧实现）。

#### 6.6.2 无人机映射

在执行面板中，需要将阵列中的无人机槽位与真实注册无人机一一对应：

- **手动连线**：在面板中拖拽连线，将阵列槽位与注册无人机对应
- **一键自动连接**：系统按注册顺序自动分配（槽位 1 → 注册 ID 最小的在线无人机，以此类推）
- 映射完成后显示对应关系预览，用户确认后方可执行

#### 6.6.3 阵列执行中的暂停

- 按 `P` 键：
  - 若当前镜头为**跟随镜头**（跟随某架无人机）：仅暂停该无人机
  - 若当前镜头为**自由镜头**：暂停所有参与阵列的无人机
- 暂停后无人机悬停在当前位置，再次按 `P` 或点击"继续"按钮恢复

### 6.7 鼠标中键查看无人机信息

- 鼠标中键点击无人机（实时控制和阵列执行模式下均可用）
- 弹出该无人机的信息面板，显示：
  - 无人机名称、ID、型号
  - 在线状态
  - 当前位置（GPS 坐标 + UE 世界坐标）
  - 当前高度（米）
  - 当前速度（m/s）
  - 当前姿态（俯仰/横滚/偏航）
  - 当前任务状态（空闲 / 执行中 / 暂停 / 失联）
  - 影子机与镜像机位置偏差（米）
  - RTSP 视频流画面（若已配置）
- 信息面板可拖拽移动，可关闭
- 同时可打开多架无人机的信息面板

### 6.8 小地图

实时部署模块**右下角**以 UI 形式显示小地图（独立 2D 地图图层，非 Cesium 场景缩略图）：

#### 6.8.1 显示内容

- 所有已注册无人机以**圆点**标注，相对位置与俯视角观察真实位置一致
- 数据来源：**镜像机遥测数据**（与镜像机同步）
- 颜色规则：
  - 正常在线：**绿色**
  - 低电量或断联：**红色**

#### 6.8.2 告警弹窗

当检测到以下情况时，在 **UI 左下角**弹出提示弹窗：

| 触发条件 | 提示内容 |
|----------|----------|
| 无人机电量低于阈值（可配置，建议默认 20%） | "⚠ [无人机名称] 电量不足，请注意！" |
| 无人机断联（超时未收到遥测） | "⚠ [无人机名称] 已断联！" |

- 弹窗可手动关闭，也可在状态恢复后自动消失
- 同一无人机的同类告警不重复弹出（直到状态恢复后再次触发）

#### 6.8.3 交互

- 小地图可缩放（滚轮）
- 点击小地图上的无人机点，主视角切换到该无人机的跟随镜头

## 七、待讨论事项

1. **实时控制通讯延迟**：HTTP 轮询延迟可能不满足实时控制需求（目标点派发需要低延迟），是否在控制命令通道引入 WebSocket 或其他低延迟协议？
2. **后端管理飞控进程**：后端是否负责飞控桥接进程的启停管理，还是桥接进程独立运行、后端仅通过协议发送命令？
3. **避障算法**：实时部署中未预演阵列的避障算法在 UE 侧实现，具体算法方案（势场法、RRT 等）需确认。
4. **影子机飞行模拟**：影子机按设定速度和轨迹在 UE 中插值移动。是否需要模拟加速/减速过程（如起飞加速、到点减速），还是全程匀速直线插值即可？
5. **低电量阈值**：低电量告警的默认阈值（建议 20%），以及电量数据是否由遥测数据提供？
6. **无人机注册表单字段设计**：注册表单的具体字段（必填项、字段类型、与后端 API 的对应关系）需在后端接口设计完成后确认。需明确：后端注册接口需要哪些字段、无人机 ID 是由用户指定还是后端自动分配、是否需要端口配置（取决于后端与无人机的通信方式）。
