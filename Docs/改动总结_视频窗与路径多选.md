# 第一部分:视频流独立窗口

## 1.1 功能概述

预演关卡(CesiumWorld)提供「视频流」入口,勾选无人机后为每台 `video_url` 非空的无人机创建一个 **Windows 系统级独立顶层窗口**(可拖到第二显示器),窗口内用内置浏览器(CEF)播放 MediaMTX 的 WebRTC 页面。最多同时 4 路,超出拒绝并提示。

数据流(接后端时):
- 控制链路:后端 `GET /api/drones` 返回每台的 `video_url` 字符串 → 写入无人机注册表(`DroneNetworkManager.cpp` 解析 `video_url`)。
- 媒体链路:UE 的 WebBrowser 直连 MediaMTX 的 WebRTC 播放页拉流,**不经过 DroneBackend**。
- `OpenDroneVideoWindow` 只从注册表读 `Descriptor.VideoUrl`,后端无关。

## 1.2 核心文件(功能主体)

- `Source/UE5DroneControl/UI/DroneVideoWindowManager.h / .cpp`
  拥有并管理全部 `SWindow` 独立窗口。约束:同机最多一窗、最多 4 窗、四条关闭路径统一汇入 `CloseVideoWindow(DroneId)`(顺序:停视频 → 解绑回调 → 销毁窗口 → 移除记录);`SWindow` 与 `UUserWidget` 强引用防 GC。
- `Source/UE5DroneControl/UI/DroneVideoWindowWidget.h / .cpp`
  单个窗口的内容控件:WebBrowser 视频生命周期、名称/ID、全屏/恢复/关闭按钮,不查 DroneBackend。
- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.h / .cpp`
  对外蓝图入口:`OpenDroneVideoWindow` / `CloseDroneVideoWindow` / `CloseAllDroneVideoWindows` / `HasDroneVideoWindow`;持有 manager(强引用防 GC);`EndPlay` 关闭全部窗口。

## 1.3 本轮修复 A:视频加载失败 → 可重试错误状态

**文件:`DroneVideoWindowWidget.h / .cpp`**

背景:UMG 的 `UWebBrowser` 只暴露 `OnUrlChanged`,没有 `OnLoadError` 委托。MediaMTX 不可达 / URL 打不开时,窗口会卡在「正在加载视频…」无法恢复。

改动:
- 新增 `VideoLoadTimeoutSeconds`(默认 10 秒,`EditAnywhere`)、`bLoadConfirmed`、`VideoLoadTimeoutHandle`、`HandleVideoLoadTimeout()`。
- `StartVideo()`:加载时启动**看门狗计时器**。
- `OnVideoUrlChanged()`:
  - 成功导航到真实页 → 取消计时器、隐藏覆盖层、置 `bLoadConfirmed`。
  - 命中 CEF 报错页(`chrome-error` / `chromewebdata`)→ 立即显示「视频加载失败,请点重试」+ 重试按钮。
- 计时器超时仍未确认 → 同样显示可重试错误状态。
- `StopVideo()` / `ShutdownWidget()` / `RetryVideo()` 均正确清理或重置计时器。
- 新增 include:`Engine/TimerHandle.h`(头);`Engine/World.h`、`TimerManager.h`(源)。

## 1.4 本轮修复 B:任意方式关窗 → 通知蓝图(复选框自动弹回)

背景:用标题栏 ×(或窗内关闭按钮)关窗时,选择面板对应行的复选框不同步,导致重开需点两次。

**`DroneVideoWindowManager.h / .cpp`**
- 新增原生委托 `FOnVideoWindowClosedNative`(带 `DroneId`)与成员 `OnWindowClosed`。
- 在统一清理出口 `CloseVideoWindow()` 末尾(记录移除、窗口销毁之后)`OnWindowClosed.Broadcast(DroneId)`。
- 覆盖全部四条关闭路径:原生标题栏 ×、窗内 UMG 关闭按钮、取消勾选、离开关卡/退出。

**`DroneOpsPlayerController.h / .cpp`**
- 新增蓝图可绑定动态委托 `FOnDroneVideoWindowClosed`(带 `DroneId`)与 `UPROPERTY(BlueprintAssignable)` 属性 `OnDroneVideoWindowClosed`。
- 新增私有桥接 `HandleVideoWindowClosed(int32)`。
- `EnsureVideoWindowManager()` 订阅 manager 的 `OnWindowClosed` → 转发到蓝图事件。

事件链:任意关窗 → `CloseVideoWindow` → `manager.OnWindowClosed.Broadcast` → `HandleVideoWindowClosed` → `OnDroneVideoWindowClosed.Broadcast`(蓝图订阅)。

## 1.5 蓝图资产

- `WBP_DroneVideoWindow` — 窗口内容(视频 + 名称/ID + 全屏/恢复/关闭 + 状态覆盖层)。
- `WBP_VideoStreamSelector` — 视频流选择面板(读注册表生成列表)。
- `WBP_VideoStreamRow` — 列表行(复选框 + 名字;勾选=开窗、取消=关窗、拒绝=回弹)。
- `WBP_Toast` — 提示位置调整(从左上挪开,避开视频流按钮)。
- `WBP_DroneOpsHUD` — HUD 上的「视频流」入口按钮。

# 第二部分:编辑模式路径点多选 / 批量拖拽 / 撤销

## 2.1 概述

原编辑模式为单选航点。本轮增强为**多选**:可 Shift/Ctrl+点击加/减选、Ctrl+A 全选,整组按同一位移批量拖拽;并加入操作**撤销**(增点 / 删点 / 移动)。为避免多组 Gizmo 轴重叠误操作,仅「主操作航点」显示可交互 Gizmo,其余选中航点只保留高亮。

## 2.2 改动文件

**`DroneOps/Control/DroneOpsPlayerController.h / .cpp`**(多选 + 撤销主体)
- 多选集合 `EditSelectedWaypoints`;主操作航点 `EditSelectedWaypoint` 恒在集合内。
- 选择操作:`IsEditWaypointSelected` / `ToggleEditWaypointSelection` / `ClearEditWaypointSelection` / `SelectAllEditWaypoints` / `HandleSelectAllWaypointsKey`(Ctrl+A)。
- 批量拖拽:`BeginDeferredUpdateForSelection` / `EndDeferredUpdateForSelection` / `MoveSelectionAlongAxis`(整组按同一世界轴增量平移)。
- Gizmo 显示:`RefreshSelectionGizmoVisual`(仅主操作航点可交互)。
- 健壮性:`PruneInvalidSelection`(每帧剔除失效航点并迁移主选中)。
- 撤销:`EEditUndoKind`(AddWaypoints / DeleteWaypoint / MoveWaypoints)、`FEditWaypointUndoItem` / `FEditUndoEntry` / `EditUndoStack`;`PushAddWaypointsUndo` / `PushDeleteWaypointUndo` / `CaptureMoveUndoStart` / `CommitMoveUndoIfMoved` / `ClearEditUndoHistory`。

**`PathEditor/DroneWaypointActor.h / .cpp`**
- 新增 `bGizmoInteractable` + `SetGizmoInteractable` / `IsGizmoInteractable`:与选中高亮解耦,多选时仅主航点显示可交互 Gizmo 轴,避免重叠误拖。默认 true,单选行为不变。

**`PathEditor/DronePathActor.h / .cpp`**
- `InsertWaypoint(Index, WaypointData)`:供撤销「删除航点」还原用,Index 夹到 `[0, Num]`。
- `GetWaypointHandleActors()`:暴露航点句柄数组,供全选遍历。

## 2.3 交互一致性说明

- 首航点(编队锚点)固定:不可单独选中/删除,但可随编队整体拖拽。
- 冻结路径(已取消选中的无人机)航点保留可见但不可选中/拖拽。
- 播放中路径航点不可拖动。
- 以上口径统一由 `CanInteractWithEditWaypoint` 拦截,保证点选、Ctrl+点击、Ctrl+A、拖拽、删除行为一致。

# 第三部分:⚠️ 编译要求

本轮改了头文件并新增了 `UPROPERTY` / `BlueprintAssignable` 委托,**不能用 Live Coding(Ctrl+Alt+F11)**。必须:

1. 关闭 UE 编辑器。

3. Build 成功后重新打开编辑器。
