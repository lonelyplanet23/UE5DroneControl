# 预演与控制扩展需求说明

> 版本：v1.0  
> 日期：2026-07-20  
> 适用范围：`CesiumWorld` 预演关卡、无人机状态面板、路径编辑器、UE 网络层与 `BackEnd/` 接口文档。  
> 本文用于分配给其他程序员实施；除本文标为“待确认”的项外，不得自行改变交互或协议语义。

## 一、已冻结的共识

1. **刷新**统一称为 `refresh`：UE 点击后调用 `POST /api/drones/refresh`，要求后端对断连无人机发送安全 hold 探测；这不是强制重启全部连接。
2. 视频必须是 **Windows 系统级独立窗口**，可拖到第二显示器；不是 UE 主视口内的浮层。
3. 批量坐标元组固定顺序为 **`(经度,纬度,海拔)`**，即 `(longitude, latitude, altitude_msl_meters)`。
4. CesiumWorld 的“纯本地预演模式”是 **严格隔离**：开启后 UE 不得向 DroneBackend 发起任何新的 HTTP 或 WebSocket 请求。
5. 第 3 项的“多个点”在本期用于路径编辑：依次加入当前编辑的每条路径。非编辑状态仍保留既有单点派发语义，批量输入不得被擅自转换成连续真实飞行指令。

## 二、统一约束

- 改动必须兼容当前 UE 5.7 Win64 打包目标；不得只在 PIE 或编辑器中验证。
- 所有后端控制、HTTP 写操作和 WebSocket 控制消息必须经 `UDroneNetworkManager` 的统一入口，不得在 UI、Pawn 或 PlayerController 中新增绕过网络层的直发代码。
- 严格本地预演模式开启时，UI 必须明确显示“后端通信已隔离”；被拦截操作必须返回失败/已拦截提示，不能在本地伪造后端成功回调。
- 视频窗口与路径编辑都必须在离开 CesiumWorld、全局停止或程序退出时清理资源，不能残留原生窗口、浏览器实例、Timer、Actor 或悬空委托。
- 本次新增/补齐的接口约束以《接口与通讯数据规范.md》v2.2 为准。

## 需求 1：全机状态面板的选中与 refresh

### 核心需求

1. 在全机状态面板的每架无人机条目增加“选中”按钮。
2. 按钮文案和状态必须反映真实选择状态：未选中时显示“选中”，点击后加入多选集合并显示“取消选中”；已选中时点击后移出多选集合。
3. 面板增加一个全局 `refresh` 按钮。点击后 UE 调用 `POST /api/drones/refresh`，并依据响应展示刷新结果。
4. `refresh` 只请求后端探测 `offline/lost` 无人机；响应成功不等于无人机立刻在线。UI 必须如实展示“已请求探测 N 架”，后续在线状态仍以遥测/注册表更新为准。
5. 严格本地预演模式开启时，`refresh` 按钮禁用，并明确提示“纯本地预演模式：已禁止请求后端 refresh”。

### 重点阅读代码

- `Source/UE5DroneControl/UI/DroneListWidget.h/.cpp`：全机状态面板构建、注册表刷新与条目创建。
- `Source/UE5DroneControl/UI/DroneListItemWidget.h/.cpp`：单条无人机卡片的显示与交互入口。
- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.cpp`：`HandleDroneClick` 已实现真实多机选中/取消选中逻辑。
- `Source/UE5DroneControl/DroneOps/Core/DroneRegistrySubsystem.*`：多选集合和主选中机状态。
- `Source/UE5DroneControl/DroneOps/Network/DroneNetworkManager.*`、`DroneHttpClient.*`：新增 `RefreshDroneConnections` 一类的统一 HTTP 封装。
- `BackEnd/http/http_server.cpp` 与 `BackEnd/drone/drone_manager.cpp`：现有 `POST /api/drones/refresh` 和 `RefreshDisconnectedConnections()` 的实际语义。
- `接口与通讯数据规范.md`：已补充接口定义，实施时不得改变其返回字段。

### 约束条件

- 面板按钮必须复用注册表中的选择状态，不能维护第二份“仅 UI 选中”状态。
- 离线无人机沿用场景点击规则：不可被选中；按钮应禁用或给出明确提示。
- 刷新请求进行中必须防重复点击；网络失败、非 2xx 或解析失败必须恢复按钮并显示失败原因。
- `refresh` 的响应只负责结果提示；收到响应后可立即触发一次列表刷新，但不得把 `refreshed_drone_ids` 直接写成 `online`。
- 不修改后端接口路径、请求体与响应格式。

### 验收标准

1. 面板点击某架在线无人机“选中”后，场景高亮、注册表多选集合和按钮文案同步变化；再次点击可取消选中。
2. 场景中点击无人机后，面板对应按钮也能在下一次 UI 刷新时正确反映状态。
3. 后端存在 `lost/offline` 无人机时点击 `refresh`，抓包或后端日志可确认只发出一次 `POST /api/drones/refresh`；UI 展示响应中的实际探测数量。
4. 后端返回空 `refreshed_drone_ids` 时，UI 显示“当前无断连无人机需要探测”，不报错。
5. 严格本地预演模式开启时，点击/尝试点击 `refresh` 不产生 HTTP 请求。

## 需求 2：预演关卡多无人机视频的 Windows 独立窗口

### 核心需求

1. 在 CesiumWorld 预演关卡提供“视频流”入口；点击后展开无人机列表，可勾选需要查看的视频流。
2. 每个被勾选且 `video_url` 非空的无人机创建一个 **Windows 系统级独立顶层窗口**。窗口可以由操作系统拖到第二显示器，不是主视口内的 UMG 浮窗。
3. 同一无人机同一时刻最多一个视频窗口；再次勾选已打开的无人机只激活并置前其已有窗口。
4. 每个窗口显示对应视频、无人机名称/ID、全屏/恢复按钮和关闭按钮；系统标题栏关闭也必须清理同一窗口。
5. 关闭窗口、取消勾选、离开 CesiumWorld、全局停止或退出程序时，停止并释放视频页面与原生窗口。
6. 初版最多同时播放 4 路视频；超过上限时拒绝打开并给出明确提示。

### 重点阅读代码

- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.h/.cpp`：当前只维护 `CurrentDroneInfoPanel` 的单窗口逻辑，需改为多窗口管理或迁移职责。
- `Source/UE5DroneControl/UI/DroneInfoPanelWidget.h/.cpp`：`UWebBrowser` 生命周期、`LoadURL`、`about:blank` 释放逻辑及遥测显示。
- `Docs/DroneInfoPanel_WBP_Setup.md`：现有视频控件与蓝图绑定要求。
- `Docs/MediaMTX/README.md`、`Docs/MediaMTX/mediamtx.yml`：`video_url` 必须是浏览器 WebRTC 播放页，不是原始 RTSP 或 WHEP 地址。
- `Source/UE5DroneControl/UE5DroneControl.Build.cs`：已含 `Slate`、`SlateCore`、`WebBrowserWidget`；独立窗口实现复用这些模块。
- Unreal Slate：使用 `SWindow` + `FSlateApplication::AddWindow` 创建顶层窗口；窗口内容可承载从 `UUserWidget::TakeWidget()` 得到的 Slate 内容。

### 约束条件

- 使用 `FSlateApplication::AddWindow` 创建独立顶层窗口；不要用 `AddToViewport` 冒充系统窗口，也不要改成外部浏览器进程。
- 每个窗口须保存 `TSharedPtr<SWindow>` 与对应 `UUserWidget` 的强引用，避免 Slate 窗口和 UE UObject 生命周期不一致或被 GC。
- 原生窗口关闭回调、UMG 关闭按钮、取消勾选和关卡退出必须汇入同一个 `CloseVideoWindow(DroneId)` 清理函数；清理顺序为停止视频/导航 `about:blank`、解绑回调、销毁窗口、移除记录。
- “全屏”指该视频独立窗口在它当前所在显示器上的原生全屏/恢复，不改变 UE 主游戏窗口；恢复时应还原用户进入全屏前的窗口大小与位置。
- `video_url` 为空、视频页加载失败、MediaMTX 不可达时，不创建无内容窗口或必须在窗口内显示可重试错误状态。
- 严格本地预演模式不得为了刷新视频列表访问 DroneBackend；视频选择数据只能使用进入隔离前已缓存的无人机注册表。直接访问独立 MediaMTX 视频地址不属于 DroneBackend 控制链路。

### 验收标准

1. 勾选两架已配置视频地址的无人机后，Windows 任务栏/Alt+Tab 可见两个独立窗口；任一窗口可拖到第二显示器且视频持续播放。
2. 重复勾选同一无人机不创建第二个窗口，而是将既有窗口置前。
3. 每个窗口的全屏/恢复按钮可用，恢复后仍回到进入全屏前的屏幕与尺寸；关闭按钮和 Windows 标题栏 `X` 都能关闭窗口。
4. 关闭一个窗口不影响其他视频窗口与 UE 主窗口；再次勾选该机可重新建立视频连接。
5. 连续打开第 5 路视频时被拒绝，并显示“最多同时播放 4 路视频”。
6. Win64 打包产物上完成双屏测试：拖屏、混合 DPI、全屏/恢复、最小化主窗口、关闭视频窗、退出程序后均无残留窗口或崩溃。

## 需求 3：批量坐标输入并添加路径点

### 核心需求

1. 坐标输入面板新增“批量坐标”输入框，接受连续元组：`(经度,纬度,海拔)(经度,纬度,海拔)`。
2. 一个输入可包含多个坐标点；允许元组之间有空格、换行，允许数字为正负数和小数。
3. 批量输入仅在路径编辑模式下用于加点：按输入顺序，为当前每一条编辑路径依次添加对应航点，并沿用现有编队相对偏移规则。
4. 非编辑模式继续使用已有的单点输入和“位置预览/派发”行为；批量输入框不可用于把多个点自动派发给真实无人机。

### 重点阅读代码

- `Source/UE5DroneControl/UI/GeographicTargetPanelWidget.h/.cpp`：当前经度、纬度、海拔输入、校验、预览和派发入口。
- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.h/.cpp`：`AddGeographicWaypointInEditMode`、`TryConvertGeographicToWorld`、`AddWaypointToAllEditingPaths`。
- `Source/UE5DroneControl/PathEditor/DronePathActor.*`、`DroneWaypointTypes.h`：航点数据、路径刷新和保存格式。
- `坐标系转换说明.md`：WGS84、海拔与 UE/Cesium 世界坐标转换约定。

### 约束条件

- 元组字段顺序必须是 **经度、纬度、海拔**，不得沿用其他界面的经纬顺序或在内部静默交换。
- 解析必须整串匹配：不允许跳过不合法字符或只执行部分合法元组。任一元组格式、数值或范围错误时，整批不得改动任何路径。
- 经度范围 `[-180, 180]`，纬度范围 `[-90, 90]`；海拔使用现有 MSL 高程约定和既有可接受范围。
- 错误提示必须指出第几个元组和失败原因，例如“第 2 个坐标纬度超出 [-90, 90]”。
- 成功前先完成全部解析与坐标转换，再统一写入路径；任何一点转换失败时不得留下半批航点。
- 批量加点只改本地临时路径，不产生后端 HTTP 或 WebSocket 控制消息；后续是否派发仍由现有派发流程和严格隔离总开关决定。

### 验收标准

1. 输入 `(116.397,39.908,50)(116.398,39.909,52)` 并执行后，每条当前编辑路径按顺序增加 2 个航点。
2. 允许 `(116.397, 39.908, 50)\n(116.398,39.909,52)`，结果与无空格输入一致。
3. 输入缺少括号、字段不足、非数字、经纬越界时，显示具体第几组错误，且所有路径航点数量与位置完全不变。
4. 不在路径编辑模式时，批量输入操作明确提示“批量坐标仅用于路径编辑”，不向后端发送移动或阵列指令。
5. 多机编辑时，新增点保持既有编队相对偏移；保存、播放和最终派发读取到的航点顺序正确。

## 需求 4：路径点多选与同步位置调整

### 核心需求

1. 在 CesiumWorld 路径编辑模式中，航点支持多选。
2. 普通点击一个航点时，清空旧选择并只选中该航点；按住 `Shift` 或 `Ctrl` 点击航点时，追加/取消该航点的选中状态。
3. 拖动任一已选航点的 X/Y/Z Gizmo 轴时，全部选中航点沿同一世界坐标轴移动相同距离，保持它们之间的相对位置不变。
4. 仅本期实现“多选后同步调整位置”；批量删除、批量改速度和批量改等待时间不在本期范围内，既有单点删除行为保持不变。

### 重点阅读代码

- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.h/.cpp`：`EditSelectedWaypoint`、`HandleEditModePressed`、`UpdateDraggedEditWaypoint`、`HandleEditModeReleased`、路径编辑输入处理。
- `Source/UE5DroneControl/PathEditor/DroneWaypointActor.h/.cpp`：选中显示、Gizmo 轴命中、`BeginDeferredPathUpdate`、`MoveAlongGizmoAxis`、`EndDeferredPathUpdate`。
- `Source/UE5DroneControl/PathEditor/DronePathActor.*`：航点位置写回、路径刷新和播放中冻结规则。
- `Source/UE5DroneControl/UI/SequenceDispatchPanelWidget.*`：路径编辑模式的进入、退出、保存、播放与清理生命周期。

### 约束条件

- 将单个 `EditSelectedWaypoint` 扩展为“主操作航点 + 已选航点集合”；集合必须过滤无效 Actor、已销毁 Actor、播放中路径和被冻结路径的航点。
- 所有被移动航点在拖动开始时进入 deferred update；鼠标释放时再分别提交路径数据和刷新路径，避免每帧重复重建路径。
- 多选时仅主操作航点显示可交互 Gizmo 轴，其他选中航点只显示选中高亮，避免多组 Gizmo 重叠造成误操作。
- 若用户点击的 Gizmo 所属航点尚未选中，先按普通单选规则选中它，再开始拖动。
- 组拖动必须使用同一个位移增量，不得把所有航点强行移动到同一绝对位置。
- 离开编辑模式、全局停止、删除主航点或路径销毁时，必须清理选中集合与拖拽状态，不能保留悬空引用。
- 本需求只编辑本地路径；不会直接向后端发送指令，最终派发仍受需求 5 的统一隔离控制。

### 验收标准

1. 普通点击航点 A 后只选中 A；Shift/Ctrl 点击 B 后 A、B 同时高亮；再次 Shift/Ctrl 点击 A 后仅 B 保持选中。
2. 选中不同路径上的 A、B 后拖动 A 的 X 轴 100 cm，A、B 的 X 坐标均增加 100 cm，二者原有相对距离保持不变。
3. 组拖动完成后，每条所属路径的连线、保存数据、再次进入编辑模式后的航点位置均正确。
4. 播放中或冻结路径的航点不能被加入可拖动选择集合，也不会因拖动其他点而被改变。
5. 退出编辑模式、全局停止或删除选中主点后，不再出现残留高亮、Gizmo 或崩溃。

## 需求 5：CesiumWorld 纯本地预演严格隔离 Toggle

### 核心需求

1. 在 CesiumWorld 中增加一个视觉优先级高、始终可见的 Toggle：`纯本地预演（隔离后端）`。
2. 默认关闭。开启后立即进入严格隔离：UE 不得再向 DroneBackend 发起任何新的 HTTP 请求或 WebSocket 消息。
3. 开启时停止 `GET /api/drones` 轮询、断开 WebSocket、取消仍在进行中的 HTTP 请求，并在所有被拦截的操作处显示明确原因。
4. 关闭后才可恢复与后端的通信：重新建立 WebSocket、恢复轮询、立即执行一次 `GET /api/drones` 同步注册表。
5. 离开 CesiumWorld 时自动清理并复位 Toggle 为关闭，避免状态泄漏到其他关卡。

### 重点阅读代码

- `Source/UE5DroneControl/DroneOps/Network/DroneNetworkManager.h/.cpp`：初始化、轮询、WebSocket 生命周期、`SendMoveCommand`、`SendPauseCommand`、注册与阵列派发入口。
- `Source/UE5DroneControl/DroneOps/Network/DroneHttpClient.h/.cpp`：所有 HTTP 请求创建、回调与取消能力；需要补齐可追踪/取消在途请求的机制。
- `Source/UE5DroneControl/DroneOps/Network/DroneWebSocketClient.h/.cpp`：连接、断开、发送消息。
- `Source/UE5DroneControl/DroneOps/Control/DroneOpsPlayerController.cpp`、`MultiDroneCharacter.cpp`：移动控制调用点。
- `Source/UE5DroneControl/UI/SequenceDispatchPanelWidget.cpp`：阵列派发、路径编辑派发和本地预演的调用点。
- `Source/UE5DroneControl/UI/DroneListWidget.*`：需求 1 的 `refresh` 调用点。
- `接口与通讯数据规范.md` 第 1.1 节：严格隔离的协议边界。

### 约束条件

- 隔离开关状态必须由 `UDroneNetworkManager` 或等价的 GameInstance 级服务保存；禁止由单个 Widget 私有布尔值决定是否发包。
- `UDroneNetworkManager` 必须提供统一的 `CanSendToBackend` / `IsStrictLocalPreviewIsolation` 判断，且覆盖：移动、暂停、恢复、无人机注册/编辑/删除、`refresh`、阵列创建/停止、debug 请求和后续新增出站操作。
- 新增或重构控制代码不得直接调用 `UDroneHttpClient` 或 `UDroneWebSocketClient::SendMessage` 绕过总开关。
- 开启后的“严格”含义是：不发起新 HTTP（包括 `GET /api/drones`）和 WebSocket 控制消息，并主动断开现有 WebSocket；已抵达后端的旧请求无法撤回，需在 UI/日志中如实说明。
- 严格隔离期间允许本地影子机播放、路径编辑、坐标预览和本地 UI；允许直接加载独立 MediaMTX 视频 URL，但不得借此调用 DroneBackend API。
- 所有拦截路径都必须执行回调或返回结构化失败，避免调用方永久等待“派发中”。
- Toggle 开启时应有醒目的持续状态标识，且不能被普通面板遮挡。

### 验收标准

1. 开启 Toggle 后，网络日志/抓包确认停止 `GET /api/drones` 轮询，WebSocket 已断开，之后不再出现 UE→DroneBackend 的 HTTP 或 WebSocket 出站请求。
2. 在隔离状态下尝试移动无人机、暂停/恢复、派发阵列、编辑路径后派发、点击 `refresh`，均被本地拒绝且 UI 提示“纯本地预演模式已阻止后端通信”；后端日志无对应请求。
3. 隔离状态下路径编辑、批量坐标加点、影子机本地播放和本地视频窗口仍可正常使用。
4. 关闭 Toggle 后，UE 恢复 WebSocket，恢复定时轮询并立即成功请求一次 `GET /api/drones`；之后控制操作可正常发送。
5. 在 HTTP 请求进行中开启 Toggle，未发送完成的可取消请求被取消，调用方收到失败/已取消结果，不会卡在加载状态。
6. 切换关卡或退出 CesiumWorld 后，隔离状态、Timer、请求追踪和 UI 提示全部正确清理；再次进入 CesiumWorld 时 Toggle 默认为关闭。
