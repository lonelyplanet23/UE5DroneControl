# 室内/室外视频与 SfM 元数据录制测试手册

> 适用当前环境：Jetson `192.168.10.1`，MediaMTX、后端和 UE 暂时运行在 Windows `192.168.10.30`，DroneId 为 `1`。
>
> 本手册只验证摄像头、视频传输、Windows 录像和逐帧 GPS/位姿元数据，不把室内测试当作飞行测试。

## 1. 测试完成后应该得到什么

Windows 上应出现两类文件：

```text
D:\DroneData\recordings\drone-1\...mp4
D:\DroneData\metadata\drone-1\mission_...\
├── session.json
├── completed.json
└── batches\*.jsonl
```

- `recordings`：MediaMTX 保存的无音频 H.264/fMP4 视频。
- `metadata`：后端保存的逐帧 GPS、局部位姿、图像时间和视频 PTS。
- Jetson 默认不保存 MKV 或 CSV，也不会每次创建本地任务目录。

## 2. 测试前安全要求

### 室内测试

- [ ] 拆下桨叶，或完全断开电机动力。
- [ ] 将无人机固定在稳定台面，不在人员、玻璃或易损物旁测试。
- [ ] 本次只移动摄像头或缓慢改变机身朝向，不执行室内起飞。
- [ ] 室内 GPS 无效属于正常现象，不要为了获得 GPS 在室内解锁或启动电机。

### 室外测试

- [ ] 使用符合当地法规和场地要求的空旷区域。
- [ ] 由具备相应能力的人员负责飞行安全；本手册不替代飞行检查单。
- [ ] 确认天气、光照、存储空间和网络覆盖满足测试要求。
- [ ] 在移动或起飞前先启动录制，落地并静止后再停止录制。

## 3. 一次性准备

### 3.1 确认本机程序和配置

项目目录：

```text
D:\RedAlert\UE5DroneControl
```

当前文件：

```text

配置文件：D:\RedAlert\UE5DroneControl\tools\MediaMTX\mediamtx.yml
后端程序：D:\RedAlert\UE5DroneControl\Backend\build\Release\DroneBackend.exe
Jetson脚本：D:\RedAlert\UE5DroneControl\Jetson\jetson_video_stream.py
```

当前实际 `tools\MediaMTX\mediamtx.yml` 已启用：

```yaml
record: true
recordPath: D:/DroneData/recordings/%path/%Y-%m-%d_%H-%M-%S-%f
recordFormat: fmp4
recordSegmentDuration: 10m
recordDeleteAfter: 0s
```

### 3.2 检查 Windows 磁盘空间

在 PowerShell 中执行：

```powershell
Get-PSDrive D | Select-Object Name,
  @{Name='UsedGB';Expression={[math]::Round($_.Used/1GB,1)}},
  @{Name='FreeGB';Expression={[math]::Round($_.Free/1GB,1)}}
```

估算空间：

| 视频码率 | 单架无人机每小时约占用 |
|---|---:|
| 8 Mbps | 3.6 GB |
| 16 Mbps | 7.2 GB |
| 20 Mbps | 9 GB |

建议测试前至少预留“预计用量的两倍”，同时考虑多架无人机并行录制。

### 3.3 复制最新版脚本到 Jetson

在 Windows 项目根目录执行：

```powershell

  jetson1@192.168.10.1:/home/jetson1/jetson_video_stream.py
```

### 3.4 Jetson 检查依赖

```bash
source /opt/ros/humble/setup.bash

gst-inspect-1.0 rtspclientsink >/dev/null && echo "rtspclientsink OK"
```

## 4. 每次测试的启动顺序

必须按下面顺序启动。

### 第一步：启动 Windows 后端

打开 PowerShell 窗口 A：

```powershell
Set-Location D:\RedAlert\UE5DroneControl
.\Backend\build\Release\DroneBackend.exe .\Backend\config.yaml
```

看到 HTTP 监听 `8080` 后，在另一个 PowerShell 中检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8080/
```

结果应包含 `status: ok`。

### 第二步：启动 MediaMTX

打开 PowerShell 窗口 B：

```powershell
Set-Location D:\RedAlert\UE5DroneControl\tools\MediaMTX
.\mediamtx.exe .\mediamtx.yml
```

检查端口：

```powershell
Test-NetConnection 127.0.0.1 -Port 8554
Test-NetConnection 127.0.0.1 -Port 8889
```

两次的 `TcpTestSucceeded` 都应为 `True`。

### 第三步：启动原有 Jetson 程序

保持以下现有程序正常运行：

- 摄像头 ROS2 节点。
- Micro XRCE Agent/PX4 ROS2 通讯。
- `jetson_bridge.py`。

视频脚本只订阅话题，不会与这些程序争抢摄像头、串口或控制 UDP 端口。

### 第四步：确认 ROS2 话题

在 Jetson 新终端执行：

```bash
source /opt/ros/humble/setup.bash
source /home/jetson1/swarm_ws/install/setup.bash

ros2 topic hz /image_raw
ros2 topic hz /fmu/out/vehicle_global_position
ros2 topic hz /fmu/out/vehicle_odometry
ros2 topic echo /camera_info --once
ros2 topic echo /fmu/out/vehicle_global_position --once
```

记录以下结果：

- [ ] `/image_raw` 持续输出，帧率接近摄像头设定值。
- [ ] `/camera_info` 的宽高符合当前摄像头。
- [ ] `/camera_info` 中 `k[0]` 不为 `0`；为 `0` 表示相机尚未标定。
- [ ] GPS 话题持续输出；室内允许 `lat_lon_valid=false`。
- [ ] 里程计话题持续输出。

### 第五步：启动 Jetson 视频脚本

先创建一个只保存日志的小目录：

```bash
mkdir -p /home/jetson1/video_test_logs
```

当前摄像头或室内首次测试使用 8 Mbps：

```bash
python3 /home/jetson1/jetson_video_stream.py \
  --drone-id 1 \
  --jetson-ip 192.168.10.1 \
  --mediamtx-host 192.168.10.30 \
  --backend-base-url http://192.168.10.30:8080 \
  --bitrate 8000000 \
  2>&1 | tee /home/jetson1/video_test_logs/indoor_$(date +%Y%m%d_%H%M%S).log
```

未来确认摄像头实际输出为 1080p/30fps 后，室外 SfM 测试可改为：

```bash
python3 /home/jetson1/jetson_video_stream.py \
  --drone-id 1 \
  --jetson-ip 192.168.10.1 \
  --mediamtx-host 192.168.10.30 \
  --backend-base-url http://192.168.10.30:8080 \
  --bitrate 16000000 \
  --fps 30 \
  2>&1 | tee /home/jetson1/video_test_logs/outdoor_$(date +%Y%m%d_%H%M%S).log
```

日志每 5 秒应显示：

```text
stream=streaming
metadata_queued=0 或较小数值
metadata_uploaded=持续增加
metadata_dropped=0
```

### 第六步：检查浏览器与 UE

Windows 浏览器打开：

```text
http://192.168.10.30:8889/drone-1
```

- [ ] 浏览器能连续播放。
- [ ] UE 中键打开 Drone 1 面板后显示同一画面。
- [ ] 关闭面板后 UE 停止播放，但 MediaMTX 仍继续录像。
- [ ] 重新打开面板可以继续播放。

## 5. 室内录制测试

建议第一次录制 3～5 分钟。

### 5.1 操作步骤

1. 完成第4章全部启动步骤。
2. 保持桨叶拆除、电机不运行。
3. 让画面中同时出现静态物体和容易辨认的时间/动作变化。
4. 缓慢改变摄像头朝向，避免快速甩动造成运动模糊。
5. 观察 Jetson 日志至少3分钟。
6. 在 UE 中打开、关闭、再次打开面板各一次。
7. 记录室内 GPS 的 `lat_lon_valid`、`alt_valid`、`eph`、`epv`。

### 5.2 室内预期结果

- [ ] 视频连续，无长期冻结。
- [ ] Jetson 日志中的 `queue_drops=0`，或该数值没有持续增加。
- [ ] `metadata_dropped=0`。
- [ ] 元数据持续上传。
- [ ] 即使 `lat_lon_valid=false`，每个图像帧仍有元数据记录。
- [ ] 无效 GPS 帧的经纬度为空，而不是伪造 `0,0`。
- [ ] 本地位姿有效时，NED 位置和四元数仍被保存。

### 5.3 可选：后端短暂中断测试

只在无人机安全静置时进行：

1. 保持 MediaMTX 和 Jetson 视频脚本运行。
2. 停止 Windows 后端约10秒。
3. 观察 Jetson 的 `metadata_queued` 增加。
4. 重新启动后端。
5. 确认 `metadata_queued` 逐渐回到 `0`，`metadata_uploaded` 继续增加。
6. 确认后端批次文件没有重复帧。

默认内存缓存为3600帧，30fps时约2分钟。不要故意让后端离线超过缓存容量；队列满后会增加 `metadata_dropped`。

### 5.4 可选：视频断流重连测试

只在无人机安全静置时进行：

1. 停止 MediaMTX 约5～10秒。
2. UE 应显示视频中断，不应崩溃。
3. Jetson 日志应进入重连状态。
4. 重新启动 MediaMTX。
5. 确认浏览器和 UE 可以恢复画面。

断流期间没有到达 MediaMTX 的画面无法补录，这是预期结果。

## 6. 室外录制测试

第一次室外测试建议录制5～10分钟，先验证数据质量，再增加飞行时间。

### 6.1 出发前检查

- [ ] Windows `D:` 盘空间充足。
- [ ] 室内测试已经通过。
- [ ] 摄像头镜头清洁、固定牢靠。
- [ ] 相机标定数据有效，`camera_info.k[0] != 0`。
- [ ] Jetson、MediaMTX、后端和 UE 在测试网络中互相可达。
- [ ] 电源能够覆盖计划测试时间并留有安全余量。

### 6.2 到达室外后的 GPS 检查

在移动或起飞前执行：

```bash
ros2 topic echo /fmu/out/vehicle_global_position --once
```

至少记录：

- `lat_lon_valid`
- `alt_valid`
- `dead_reckoning`
- `lat`、`lon`、`alt`
- `eph`、`epv`

开始正式 SfM 采集前应确认：

- [ ] `lat_lon_valid=true`。
- [ ] `alt_valid=true`。
- [ ] 经纬度不再是无效占位值。
- [ ] `eph`、`epv` 满足项目对定位精度的要求。
- [ ] GPS 状态已经稳定一段时间，而不是刚刚获得定位。

GPS 是否满足飞行安全要求应由飞控和现场负责人判断，本手册只检查录制数据。

### 6.3 室外录制步骤

1. 启动 Windows 后端和 MediaMTX。
2. 启动摄像头、PX4 ROS2 通讯和 `jetson_bridge.py`。
3. 确认有效 GPS 和相机参数。
4. 启动 `jetson_video_stream.py`。
5. 保持静止约30秒，建立视频与元数据开头基准。
6. 按现场批准的方式完成测试运动或飞行。
7. 结束后保持静止约30秒。
8. 先在 Jetson 视频脚本终端按 `Ctrl+C`。
9. 等待至少5秒，让最后一批元数据上传并让 MediaMTX 完成视频封装。
10. 最后再关闭 MediaMTX 和后端。

### 6.4 室外预期结果

- [ ] 全程 `metadata_dropped=0`。
- [ ] Jetson 日志中的 `queue_drops=0`，或偶发但没有持续增长。
- [ ] GPS 有效帧占绝大多数。
- [ ] 视频文件能够正常播放到末尾。
- [ ] 快速运动部分没有长期卡顿；允许因曝光或快门造成的画质问题被单独记录。
- [ ] `completed.json` 存在。
- [ ] 录像时间范围与任务 `session.json` 时间范围一致。

## 7. 正确停止顺序

每次测试结束必须按以下顺序：

1. 停止移动/飞行，保持画面稳定。
2. Jetson 的 `jetson_video_stream.py` 按 `Ctrl+C`。
3. 确认日志出现：

```text
Video pipeline and metadata processing stopped; final upload was attempted
```

4. 等待至少5秒。
5. 检查后端任务目录出现 `completed.json`。
6. 再关闭 MediaMTX。
7. 最后关闭后端。

不要先强制关闭后端，否则最后一批元数据和完成标记可能来不及上传。

## 8. Windows 上检查录制结果

### 8.1 查看最新视频

```powershell
Get-ChildItem D:\DroneData\recordings\drone-1 -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 10 FullName,Length,LastWriteTime
```

使用 VLC 或其他支持 fMP4/H.264 的播放器检查最新视频。

如果系统安装了 FFmpeg，可以进一步检查：

```powershell
ffprobe -v error -select_streams v:0 `
  -show_entries stream=codec_name,width,height,r_frame_rate,duration `
  -of default=noprint_wrappers=1 "视频文件完整路径.mp4"
```

### 8.2 查看最新元数据任务

```powershell
$mission = Get-ChildItem D:\DroneData\metadata\drone-1 -Directory |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 1

$mission.FullName
Get-Content "$($mission.FullName)\session.json"
Test-Path "$($mission.FullName)\completed.json"
```

最后一条命令应返回 `True`。

### 8.3 抽查前三帧

```powershell
$firstBatch = Get-ChildItem "$($mission.FullName)\batches\*.jsonl" |
  Sort-Object Name |
  Select-Object -First 1

Get-Content $firstBatch.FullName -TotalCount 3
```

### 8.4 统计总帧数和有效 GPS 帧数

```powershell
$batchFiles = Get-ChildItem "$($mission.FullName)\batches\*.jsonl"
$totalFrames = ($batchFiles.FullName | Get-Content | Measure-Object -Line).Lines
$validGpsFrames = (Select-String -Path $batchFiles.FullName `
  -SimpleMatch '"lat_lon_valid":true').Count

[pscustomobject]@{
  TotalFrames = $totalFrames
  ValidGpsFrames = $validGpsFrames
  ValidGpsPercent = if ($totalFrames -gt 0) {
    [math]::Round($validGpsFrames * 100.0 / $totalFrames, 2)
  } else { 0 }
}
```

室内 `ValidGpsPercent` 可以为 `0`；室外应根据实际卫星环境和项目标准判断。

### 8.5 检查失败或未送入编码器的帧

```powershell
$notAccepted = (Select-String -Path $batchFiles.FullName `
  -SimpleMatch '"stream_accepted":false').Count

$notAccepted
```

如果数值较大，应结合 Jetson 日志检查网络、编码器和 `queue_drops`。

## 10. 常见问题

### MediaMTX 启动后没有录像文件

只有 Jetson 成功发布 `drone-1` 流后才会创建录像。检查 MediaMTX 日志是否出现发布者连接，并确认实际启动的是：

```text
D:\RedAlert\UE5DroneControl\tools\MediaMTX\mediamtx.yml
```

其中必须包含 `record: true`。

### 浏览器有画面但后端没有元数据

检查 Jetson 启动参数的：

```text
--backend-base-url http://192.168.10.30:8080
```

再从 Jetson 测试：

```bash
curl http://192.168.10.30:8080/
```

### 后端有元数据但没有 `completed.json`

通常表示 Jetson 脚本被强制终止、后端提前关闭或网络中断。普通测试应使用 `Ctrl+C`，并在关闭后端前等待最后一批上传。

### 室内经纬度为空

这是没有有效 GPS 定位时的正确行为。只要帧记录仍持续生成、`lat_lon_valid=false`，就不算错误。

### 室外 GPS 有效但 SfM 效果不好

优先检查相机标定、曝光、快门、运动模糊、画面重叠率、滚动快门和时间同步。GPS 只能提供位置先验，不能替代清晰且有足够重叠的图像。
