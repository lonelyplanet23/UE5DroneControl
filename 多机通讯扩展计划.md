# 多机通讯扩展计划

## 背景

当前项目为单机 UE5+PX4 数字孪生，端口固定（8888接收/8889发送）。需扩展为多机，每架无人机一对端口，玩家可切换控制目标。

**原则：不修改现有文件，新建文件支持多机。**

---

## 端口分配方案

| 无人机 | PX4→UE5（接收） | UE5→PX4（发送） |
|--------|----------------|----------------|
| UAV1   | 8888           | 8889           |
| UAV2   | 8890           | 8891           |
| UAV3   | 8892           | 8893           |
| UAV-N  | 8888 + 2*(N-1) | 8889 + 2*(N-1) |

## 新建文件清单

### UE5 C++ 端（新建，不改现有文件）

#### 1. `Source/UE5DroneControl/MultiDroneManager.h/.cpp`
新建一个 Actor 类 `AMultiDroneManager`，放在场景中，负责：
- 持有所有 `ARealTimeDroneReceiver` 实例的引用数组（按 ListenPort 排序）
- 持有所有 `AUE5DroneControlCharacter` 实例的引用数组
- 暴露 `ActiveDroneIndex`（BlueprintReadWrite）
- 提供 `SwitchToDrone(int32 Index)` 函数，切换视角 + 更新 ActiveDroneIndex
- 提供 `GetActiveSender()` 返回当前激活的发送端 Character

PlayerController 通过 `GetAllActorsOfClass` 找到这个 Manager，把切换逻辑委托给它。

#### 2. `Source/UE5DroneControl/MultiDronePlayerController.h/.cpp`
继承 `AUE5DroneControlPlayerController`，新增：
- 按键 1/2/3 绑定 → 调用 Manager 的 `SwitchToDrone()`
- 按键 0 → 切回 TopDown 自由视角（保持原有行为）
- `OnSetDestinationReleased` 重写：点击时通过 Manager 找到当前激活的发送端 Character，调用其 `SetClickTargetLocation()`

> 注意：`AUE5DroneControlPlayerController` 中 `OnSetDestinationReleased` 是 `protected` 非虚函数，需要在子类中重新绑定输入或用 BlueprintNativeEvent 方式处理。实际上可以直接在新 Controller 里重新绑定 `SetDestinationClickAction` 的 Released 回调。

#### 3. `Source/UE5DroneControl/MultiDroneCharacter.h/.cpp`（可选）
继承 `AUE5DroneControlCharacter`，新增：
- `DroneIndex` UPROPERTY（标识这是第几架无人机）
- 其余逻辑完全复用父类

如果不想新建 C++ 类，也可以直接在蓝图 `BP_TopDownCharacter` 里加一个 `DroneIndex` 变量（纯蓝图方案）。

### Python 端（新建，不改现有文件）

#### 4. `multi_drone_config.yaml`（新建配置文件）

```yaml
drones:
  - name: "UAV1"
    ssh_host: "192.168.30.104"
    ssh_user: "jetson1"
    ssh_password: "123456"
    ros_topic: "/px4_1/fmu/out/vehicle_odometry"
    ue_host: "127.0.0.1"
    ue_receive_port: 8888   # drone_data_bridge 发送到 UE5 的端口

  - name: "UAV2"
    ssh_host: "192.168.30.105"
    ssh_user: "jetson2"
    ssh_password: "123456"
    ros_topic: "/px4_2/fmu/out/vehicle_odometry"
    ue_host: "127.0.0.1"
    ue_receive_port: 8890
```

#### 5. `multi_drone_data_bridge.py`（新建，替代 drone_data_bridge.py）
读取 `multi_drone_config.yaml`，为每架无人机：
- 开一个 SSH 终端跑 MicroXRCEAgent
- 开一个 SSH 终端跑 ros2 topic echo（输出到独立临时文件，如 `ros2_output_UAV1.txt`）
- 启动一个线程读取该文件并解析 YAML，通过 UDP 发到对应 `ue_receive_port`

核心逻辑复用 `drone_data_bridge.py` 中的 `ROS2DataReceiver` 类（import 进来直接用），只重写 `main()`。

#### 6. `multi_ue_to_px4_launcher.py`（可选，新建）
在 Jetson 上运行，读取配置，为每架无人机 `subprocess.Popen` 一个 `ue_to_px4_bridge.py` 实例：
```bash
python ue_to_px4_bridge.py --drone-id 1 --topic-prefix /px4_1 --udp-port 8889
python ue_to_px4_bridge.py --drone-id 2 --topic-prefix /px4_2 --udp-port 8891
```
`ue_to_px4_bridge.py` 本身不需要改动。

## UE5 场景配置（编辑器操作，无需写代码）

放置多个 Blueprint 实例：
- `BP_RealTimeDrone` × N，每个设置不同 `ListenPort`（8888, 8890, ...）
- `BP_TopDownCharacter` × N，每个设置不同 `RemotePort`（8889, 8891, ...）和 `DroneIndex`（0, 1, ...）
- `BP_MultiDroneManager` × 1（新建蓝图，继承 `AMultiDroneManager`）

将 GameMode 的 PlayerController 改为 `BP_MultiDronePlayerController`。

## 交互设计

| 按键 | 行为 |
|------|------|
| `0`  | 切回自由视角（TopDown Character） |
| `1`  | 选中 UAV1，视角跟随其 RealTimeDrone |
| `2`  | 选中 UAV2，视角跟随其 RealTimeDrone |
| `3`  | 选中 UAV3（如有） |
| 左键点击 | 向当前选中无人机发送目标点 |
| 空格 | 切换当前视角的俯仰角（原有行为） |

HUD 显示（蓝图 Widget）：读取 Manager 的 `ActiveDroneIndex`，显示"当前控制：UAV1"。

## 实施顺序

1. 新建 `multi_drone_config.yaml`
2. 新建 `multi_drone_data_bridge.py`（Python 端先跑通）
3. 新建 `MultiDroneManager.h/.cpp`（UE5 管理器）
4. 新建 `MultiDronePlayerController.h/.cpp`（UE5 控制器）
5. 新建 `MultiDroneCharacter.h/.cpp`（可选，也可纯蓝图）
6. 编辑器中配置场景 + GameMode
7. 测试单机兼容性（原有 BP_RealTimeDrone 端口 8888 不变）
