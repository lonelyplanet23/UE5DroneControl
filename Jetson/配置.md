可以同时运行。我检查了当前 [jetson_bridge.py](D:/RedAlert/UE5DroneControl/Jetson/jetson_bridge.py:277)：它负责 PX4 控制、遥测和 UDP 通信。新的采集程序会是独立 ROS 2 节点，只订阅已有话题，不重复打开摄像头或PX4串口。

```text
v4l2_camera
    └─ /image_raw ───────────→ 新采集程序

PX4 / MicroXRCEAgent
    ├─ GPS、姿态 ────────────→ jetson_bridge.py
    └─ 同样的话题 ───────────→ 新采集程序
```

ROS 2 允许多个节点同时订阅同一话题，因此不会冲突。

需要避免：

- 新程序不要再次打开 `/dev/video0`
- 不要再启动一个 `v4l2_camera`
- 不要再次启动 MicroXRCEAgent
- 不要占用 `jetson_bridge.py` 使用的 UDP 端口
- 新程序只读 PX4 输出话题，不发布飞控指令

## 推荐安装 rtspclientsink

你的正式推流方案优先使用 GStreamer，因为 Orin NX 的 `nvv4l2h264enc` 硬件编码器已经可用。

在 Jetson 执行：

```bash
sudo apt update
sudo apt install -y gstreamer1.0-rtsp
```

Ubuntu 22.04 ARM64 提供这个包，而 `rtspclientsink` 属于 GStreamer RTSP Server Library。[Ubuntu gstreamer1.0-rtsp](https://packages.ubuntu.com/jammy/arm64/gstreamer1.0-rtsp)、[GStreamer rtspclientsink](https://gstreamer.freedesktop.org/documentation/rtspclientsink/index.html)

安装后验证：

```bash
gst-inspect-1.0 rtspclientsink
gst-inspect-1.0 nvv4l2h264enc
```

两条命令都应输出插件信息。

如果提示找不到软件包：

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository universe
sudo apt update
sudo apt install -y gstreamer1.0-rtsp
```

## 安装 FFmpeg

FFmpeg建议也安装，主要用于检查录像、提取SfM图片和测试视频：

```bash
sudo apt update
sudo apt install -y ffmpeg
```

验证：

```bash
ffmpeg -version
ffmpeg -hide_banner -encoders | grep -Ei 'h264|nvenc|v4l2m2m|nvmpi'
```

Ubuntu 22.04 官方仓库提供 ARM64 FFmpeg。[Ubuntu FFmpeg](https://packages.ubuntu.com/en/jammy/ffmpeg)

但需要注意：普通 Ubuntu FFmpeg 不一定支持 Orin NX 的硬件编码。因此：

- 实时1080p推流：使用 GStreamer + `nvv4l2h264enc`
- 视频检查、帧提取：使用 FFmpeg
- 不建议正式直播使用 FFmpeg 软件编码

## 安装后测试推流

先在 Windows 上启动 MediaMTX，然后在 Jetson 修改下面的 IP：

```bash
MEDIAMTX_IP=192.168.10.100
DRONE_ID=d1
```

启动测试画面：

```bash
gst-launch-1.0 -e \
  rtspclientsink name=rtsp_sink \
  location="rtsp://${MEDIAMTX_IP}:8554/${DRONE_ID}" \
  protocols=tcp \
  videotestsrc is-live=true pattern=smpte \
  ! video/x-raw,width=1280,height=720,framerate=30/1 \
  ! nvvidconv \
  ! 'video/x-raw(memory:NVMM),format=NV12' \
  ! nvv4l2h264enc bitrate=4000000 insert-sps-pps=true \
    idrinterval=30 iframeinterval=30 \
  ! h264parse config-interval=-1 \
  ! queue \
  ! rtsp_sink.sink_0
```

MediaMTX官方也推荐通过 `rtspclientsink` 发布RTSP流。[MediaMTX GStreamer推流说明](https://mediamtx.org/docs/publish/gstreamer)

然后在浏览器打开：

```text
http://192.168.10.100:8889/d1
```

把 IP 换成 MediaMTX 所在 Windows 电脑的真实局域网IP。
