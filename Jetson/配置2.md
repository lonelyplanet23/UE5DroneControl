## 发现的问题

1. 启动了两个相机节点

输出中有两套 `v4l2_camera`：

```text
2623 / 2624
4702 / 4703
```

它们同时争用 `/dev/video0`，导致：

```text
/image_raw Publisher count: 0
```

2. PX4没有向ROS发布数据

虽然 `MicroXRCEAgent` 和 `jetson_bridge_1` 正在运行，但：

```text
vehicle_global_position Publisher count: 0
vehicle_odometry Publisher count: 0
```

`jetson_bridge_1` 只是订阅者，它存在不代表PX4已经连接成功。

3. 当前终端没有加载 `px4_msgs`

所以出现：

```text
Unknown package 'px4_msgs'
The message type ... is invalid
```

## 第一步：重新启动唯一的相机节点

停止现有两个相机实例：

```bash
pkill -f v4l2_camera_node
```

确认已经停止：

```bash
pgrep -af v4l2_camera
```

然后只启动一次：

```bash
source /opt/ros/humble/setup.bash

ros2 run v4l2_camera v4l2_camera_node \
  --ros-args \
  -p video_device:=/dev/video0
```

保持这个终端运行，在另一个终端检查：

```bash
source /opt/ros/humble/setup.bash
ros2 topic info /image_raw --verbose
ros2 topic hz /image_raw
```

应该看到：

```text
Publisher count: 1
average rate: 接近30
```

注意：一个相机节点可能显示一条 `ros2 run` 包装进程和一条真正的节点进程，这是正常的；之前输出里是两套，共四条进程。

## 第二步：恢复PX4发布

检查：

```bash
ros2 topic info /fmu/out/vehicle_global_position --verbose
ros2 topic info /fmu/out/vehicle_odometry --verbose
```

两者都应该显示：

```text
Publisher count: 1
```

如果还是0，在PX4的QGroundControl MAVLink Console中执行：

```text
uxrce_dds_client status
```

同时观察运行这条命令的终端：

```bash
MicroXRCEAgent serial --dev /dev/ttyTHS1 -b 921600
```

终端里应该出现PX4客户端连接信息。需要检查：

- PX4端uXRCE-DDS客户端是否启动
- PX4使用的串口是否与Jetson `/dev/ttyTHS1` 对应
- 两端波特率是否都是 `921600`
- 串口线是否连接正常

暂时不要盲目执行PX4客户端的启动命令，因为具体串口名称需要根据飞控接线确定。
