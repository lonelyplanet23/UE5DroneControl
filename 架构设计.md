### 2.2 初始化位置同步（锚定流程）

```
UE5 端触发锚定：
  1. 后端推送"无人机上电"事件（WebSocket），事件直接携带 GPS 坐标（gps_lat/gps_lon/gps_alt）
  2. UE 收到 power_on 事件，调用 Cesium API 将 GPS 转换为 UE 世界坐标 → 设为镜像机锚点
  3. UE 设置影子机初始位置 = 镜像机位置 = 锚点位置
  4. 后续每帧：镜像机位置 = 锚点 + 后端推送的 UE 坐标偏移

断联重连：
  - 后端推送 reconnect 事件（同样携带新 GPS 坐标）→ UE 重新执行步骤 2~4
```

> 说明：power_on 和 reconnect 事件均直接携带 GPS 坐标，UE5 无需额外 HTTP GET /anchor。
> GET /anchor 接口保留，仅用于 UE5 重启后补偿查询已上电无人机的锚点。

## 三、后端职责

### 3.1 各模块职责

```
后端服务
│
├── 注册管理模块
│   ├── 无人机注册（持久化：名称/ID/型号/IP:Port/视频流地址）
│   ├── 无人机删除/更新
│   └── ID ↔ 编号映射管理（ID 由后端自动分配，编号固定对应通讯端口）
│
├── 队列管理模块
│   ├── 每架无人机独立指令队列
│   ├── 心跳维持（≥2Hz 向 PX4 发送保活包，防止退出 Offboard 模式）
│   ├── 连接状态机（在线/离线/连接中/失联）
│   └── 超时检测（默认 10s 无心跳 → 失联）
│
├── 执行引擎（核心业务逻辑）
│   ├── 集结流程（阵列执行前置步骤）
│   │   ├── 接收 POST /api/arrays 后进入 ASSEMBLING 状态
│   │   ├── 为每架无人机计算初始槽位目标点（NED）
│   │   ├── 实时避障：监控所有机当前位置，动态调整集结路径防止碰撞
│   │   ├── 判断每架无人机到达初始槽位（误差 < 阈值）
│   │   ├── 全部到位后进入 READY 状态，推送 assembly_complete 事件给 UE5
│   │   └── 集结超时（默认 60s）则推送 assembly_timeout 告警
│   ├── 侦察模式：按航点序列依次发送目标点，到达后发下一个
│   ├── 巡逻模式：按航点序列飞行 + 目标识别事件触发脱离序列
│   ├── 攻击模式：经过中间航点去攻击目标点
│   ├── 阵列任务调度（多机并发执行，各自独立队列）
│   ├── 暂停/恢复控制（P 键指令处理）
│   └── 实时避障算法（集结阶段 + 实时控制阶段均启用，基于所有机当前遥测位置）
│
├── 通讯模块
│   ├── 对 UE 端
│   │   ├── HTTP REST Server（注册/查询/删除接口）
│   │   └── WebSocket Server
│   │       ├── 接收：控制指令（目标点/暂停/阵列任务）
│   │       └── 推送：遥测数据（已转换为 UE 世界坐标）、状态事件、告警
│   └── 对无人机端
│       ├── UDP 发送（24字节控制包，≥2Hz 心跳）
│       └── UDP 接收（YAML 遥测包，10Hz）
│
└── 坐标转换模块
    ├── NED → UE 偏移量转换（×100，Z 取反）
    ├── UE 偏移 → NED 控制坐标（×0.01，Z 取反）（UE5 端已预先减去锚点，后端只做单位转换）
    ├── 四元数重排与 Yaw 翻转
    └── GPS 锚点管理（上电时记录，断联重连时更新，通过 WebSocket 事件推送给 UE5）
```

### 3.2 后端 HTTP REST 接口

| 接口 | 方法 | 说明 | 调用方 |
|------|------|------|--------|
| `/api/drones` | GET | 获取所有注册无人机列表+状态 | UE 轮询（2~5s） |
| `/api/drones` | POST | 注册新无人机 | UE 注册表单提交 |
| `/api/drones/{id}` | PUT | 更新无人机信息 | UE 编辑表单提交 |
| `/api/drones/{id}` | DELETE | 删除无人机 | UE 删除操作 |
| `/api/drones/{id}/anchor` | GET | 获取无人机 GPS 锚点坐标 | UE 锚定流程 |
| `/api/arrays` | POST | 下发阵列任务 | UE 阵列执行模式 |
| `/api/arrays/{id}/stop` | POST | 停止阵列任务 | UE 停止操作 |

### 3.3 WebSocket 消息类型

**UE → 后端（控制指令）**

```json
// 目标点指令（x/y/z 为相对锚点的 UE 坐标偏移，厘米）
{ "type": "move", "drone_id": "d1", "x": 1000.0, "y": 2000.0, "z": 500.0 }

// 暂停/恢复
{ "type": "pause", "drone_ids": ["d1", "d2"] }
{ "type": "resume", "drone_ids": ["d1", "d2"] }
```

**后端 → UE（推送）**

```json
// 遥测数据（已转换为 UE 坐标偏移，单位：厘米；speed 为标量 m/s；battery=-1 表示不可用）
{ "type": "telemetry", "drone_id": "d1", "x": 10000, "y": 20000, "z": 5000,
  "yaw": 45.0, "pitch": 0.0, "roll": 0.0, "speed": 3.5, "battery": 85 }

// 状态事件（power_on 和 reconnect 必须携带 GPS 坐标）
{ "type": "event", "drone_id": "d1", "event": "power_on", "gps_lat": 39.9, "gps_lon": 116.3, "gps_alt": 50.0 }
{ "type": "event", "drone_id": "d1", "event": "reconnect", "gps_lat": 39.9, "gps_lon": 116.3, "gps_alt": 50.0 }
{ "type": "event", "drone_id": "d1", "event": "lost_connection" }

// 集结进度
{ "type": "assembling", "array_id": "a1", "ready_count": 2, "total_count": 4 }
{ "type": "assembly_complete", "array_id": "a1" }
{ "type": "assembly_timeout", "array_id": "a1", "ready_count": 2, "total_count": 4 }

// 告警
{ "type": "alert", "drone_id": "d1", "alert": "low_battery", "value": 18 }
{ "type": "alert", "drone_id": "d1", "alert": "lost_connection" }
```

## 四、前后端交互时序

### 4.1 无人机上电锚定

```
UE5                    后端（C++）              PX4无人机
 │                      │                         │
 │                      │←── UDP遥测（首包）────────│
 │                      │  记录GPS锚点              │
 │←── WS: power_on ────│  （携带gps_lat/lon/alt）  │
 │  Cesium转UE世界坐标   │                         │
 │  设置AnchorWorldLoc   │                         │
 │                      │←── UDP遥测（持续）────────│
 │←── WS: telemetry ───│  NED→UE偏移转换           │
 │  镜像机 = 锚点+偏移   │                         │
```

### 4.2 实时控制派发目标点

```
UE5                    后端（C++）              PX4无人机
 │                      │                         │
 │  用户点击地图          │                         │
 │  target - Anchor      │                         │
 │──── WS: move ────────→│                         │
 │                      │  UE偏移→NED转换           │
 │                      │──── UDP 24字节 ──────────→│
 │                      │  心跳（≥2Hz，独立线程）    │
 │                      │──── UDP 24字节 ──────────→│
 │←── WS: telemetry ───│←── UDP遥测 ─────────────│
 │  更新镜像机位置        │                         │
```

### 4.3 阵列任务执行

```
UE5                    后端（C++）              PX4无人机
 │                      │                         │
 │  用户配置路径并确认    │                         │
 │──── POST /arrays ─────→│                         │
 │                      │  进入 ASSEMBLING 状态     │
 │←── WS: assembling ───│  计算各机初始槽位          │
 │  影子机隐藏/跟随镜像机 │  实时避障路径规划          │
 │  弹窗"集结中 0/N"     │──── UDP 集结指令 ─────────→│
 │                      │←── UDP 遥测（持续）────────│
 │←── WS: telemetry ────│  判断各机是否到位          │
 │  镜像机实时渲染        │  到位计数 +1              │
 │←── WS: assembling ───│  推送进度（已到位 k/N）    │
 │  弹窗更新"集结中 k/N" │                         │
 │                      │  全部到位                 │
 │←── WS: assembly_complete│  进入 EXECUTING 状态   │
 │  影子机恢复独立运动    │  按模式逐航点发送指令      │
 │  弹窗"集结完成，执行中"│──── UDP 执行指令 ─────────→│
 │←── WS: telemetry ────│←── UDP 遥测 ─────────────│
 │  更新影子机/镜像机     │                         │
```

### 4.4 断联重连

```
UE5                    后端（C++）              PX4无人机
 │                      │  10s无遥测               │
 │←── WS: event ───────│  event=lost_connection   │
 │←── WS: alert ───────│  alert=lost_connection   │
 │  显示断联告警弹窗      │                         │
 │                      │←── UDP遥测（重连首包）────│
 │                      │  记录新GPS锚点            │
 │←── WS: event ───────│  event=reconnect          │
 │                      │  （携带新gps_lat/lon/alt）│
 │  重新Cesium转换       │                         │
 │  更新AnchorWorldLoc   │                         │
```
