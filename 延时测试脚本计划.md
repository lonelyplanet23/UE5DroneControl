# 延时测试脚本计划

## Context
项目通过 UDP 在 UE5 和 PX4 之间双向传输数据：
- **UE→PX4**：UE 发送 24 字节二进制结构体（double timestamp + 3×float + int32 mode）到端口 8889，由桥接脚本接收后转发给 PX4
- **PX4→UE**：PX4 通过 ROS2 topic echo 输出，桥接脚本解析后以 YAML 格式通过 UDP 发送到端口 8888

## 目标
创建 `latency_test.py`，测量两个方向的通讯延时：
1. **UE→PX4 方向**：从 UE 发出数据包到桥接脚本接收并处理的延时
2. **PX4→UE 方向**：从桥接脚本发出 YAML 数据到 UE 端接收的延时

## 实现方案

### 测试模式

**模式 1：UE→Bridge 延时（RTT ping-pong）**
- 启动一个本地 echo 服务器监听端口 8889（模拟桥接接收端）
- 发送方嵌入 `send_timestamp`（`time.time()`）到数据包
- Echo 服务器收到后立即原样回包到发送方的响应端口
- 计算 RTT / 2 = 单向延时
- 数据包格式复用现有 `struct "dfffI"` (24字节)，timestamp 字段即为发送时间

**模式 2：PX4→UE 延时（监听 8888 端口）**
- 监听端口 8888，接收桥接脚本发来的 YAML 数据
- 记录每个包的本地接收时间
- 从 YAML 中提取 PX4 `timestamp` 字段（微秒）
- 统计：包间隔、抖动（jitter）、丢包率、接收频率
- 注意：PX4 时钟与 Windows 时钟不同步，无法直接计算绝对延时，改为统计包间隔和抖动

**模式 3：综合报告**
- 同时运行模式 1 和模式 2
- 输出统计摘要：平均延时、最大/最小延时、标准差、丢包率

### 关键参数（与现有代码一致）
- UE→PX4 端口：8889（来自 `test_ue_sender.py` 默认值）
- PX4→UE 端口：8888（来自 `drone_data_bridge.py` 默认值）
- 数据包格式：`struct.pack("dfffI", timestamp, x, y, z, mode)`

## 脚本结构

```
latency_test.py
├── LatencyStats        # 统计计算类（RTT、jitter、丢包）
├── UEToPX4Tester       # 模式1：本地 echo 测 UE→Bridge RTT
├── PX4ToUETester       # 模式2：监听 8888 测 PX4→UE 包间隔/抖动
└── main()              # argparse: --mode [ue2px4|px42ue|both]
                        #           --count, --rate, --ue-port, --px4-port
```

## 验证
1. 运行 `python latency_test.py --mode ue2px4`（无需真实无人机，本地 echo）
2. 运行 `python latency_test.py --mode px42ue`（需要桥接脚本在运行并接收 PX4 数据）
3. 运行 `python latency_test.py --mode both` 综合测试
